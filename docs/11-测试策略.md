# 11 - 测试策略

## 11.1 测试金字塔

```
         ┌──────┐
         │ E2E  │  少量：核心流程端到端验证
         │      │  (创建 Issue → 审核 → AI 执行 → PR)
         ├──────┤
         │ 集成  │  中量：FSM 状态转换 / API 契约 / DB 交互
         │      │
         ├──────┤
         │ 单元  │  大量：纯逻辑函数 (分类器/校验器/工具函数)
         │      │
         └──────┘
```

## 11.2 单元测试

### 覆盖目标

| 包 | 目标覆盖率 | 重点测试 |

### 示例

```go
// internal/service/classifier_test.go
func TestClassify_MatchKeyword(t *testing.T) {
    tests := []struct {
        title    string
        expected string
    }{
        {"修复登录 bug", "bug"},
        {"新增用户管理功能", "feature"},
        {"升级依赖版本", "chore"},
        {"未知类型描述", ""}, // 未命中，应返回空
    }
    for _, tt := range tests {
        t.Run(tt.title, func(t *testing.T) {
            result := ClassifyByKeyword(tt.title)
            assert.Equal(t, tt.expected, result)
        })
    }
}
```

### API 契约测试

```go
// internal/handler/issue_test.go
func TestIssueAPI_CreateAndGet(t *testing.T) {
    router := setupTestRouter(t)

    // 创建
    w := performRequest(router, "POST", "/api/v1/projects/p1/issues", `{"title":"test"}`)
    assert.Equal(t, 201, w.Code)

    var resp Response
    json.Unmarshal(w.Body.Bytes(), &resp)
    issueID := resp.Data.(map[string]any)["id"].(string)

    // 查询
    w = performRequest(router, "GET", "/api/v1/issues/"+issueID, "")
    assert.Equal(t, 200, w.Code)
}
```

## 11.4 E2E 测试

### 覆盖的核心链路

```
链路 1: 需求 → Issue → 审核 → (Mock AI) → PR → 关闭
链路 2: 监控事件 → 自动去重 → 创建 Issue → AI 修复
链路 3: TestCase → 审核 → AI 生成代码 → 执行 → 结果保存
```

### 工具选型

| 工具 | 场景 |
|------|------|
| `testcontainers-go` | 启动 MySQL + Redis + Docker 容器 |
| `httptest` | HTTP 请求模拟 |
| `dockertest` | 沙盒容器生命周期管理 |

## 11.5 CI/CD 测试流水线

```yaml
# .github/workflows/test.yml
jobs:
  lint:
    runs-on: ubuntu-latest
    steps:
      - uses: golangci/golangci-lint-action@v6

  unit-test:
    runs-on: ubuntu-latest
    steps:
      - run: go test -race -coverprofile=coverage.out ./internal/...

  integration-test:
    runs-on: ubuntu-latest
    services:
      mysql:
        image: mysql:8.0
      redis:
        image: redis:7
    steps:
      - run: go test -tags=integration ./...

  coverage-check:
    needs: unit-test
    steps:
      - run: |
          coverage=$(go tool cover -func=coverage.out | grep total | awk '{print $3}')
          echo "Coverage: $coverage"
          # 阈值: 75%
```

| 检查项 | 阈值 |
|--------|:---:|
| 单元测试覆盖率 | ≥ 75% |
| 竞态检测 | 必须通过 |
| Lint (golangci-lint) | 零 error |
| 集成测试 | 全通过 |
