# 10 - 部署与运维

## 10.1 部署哲学

**极简部署，单文件启动。** 面向独立开发者和小团队，不需要 Kubernetes 集群。一个 `docker-compose up -d` 即可让整个 flowcode 栈运行起来。

## 10.2 Docker Compose 部署

### 完整 docker-compose.yml

```yaml
version: '3.8'

services:
  # MySQL 数据库
  mysql:
    image: mysql:8.0
    container_name: flowcode-db
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: ${DB_ROOT_PASSWORD:-flowcode_root}
      MYSQL_DATABASE: flowcode
      MYSQL_USER: flowcode
      MYSQL_PASSWORD: ${DB_PASSWORD:-flowcode_secret}
    volumes:
      - mysql_data:/var/lib/mysql
    ports:
      - '127.0.0.1:3306:3306'
    healthcheck:
      test: ['CMD', 'mysqladmin', 'ping', '-h', 'localhost']
      interval: 10s
      timeout: 5s
      retries: 5

  # Redis (缓存 + 任务队列)
  redis:
    image: redis:7-alpine
    container_name: flowcode-redis
    restart: unless-stopped
    command: redis-server --appendonly yes --maxmemory 256mb --maxmemory-policy allkeys-lru
    volumes:
      - redis_data:/data
    ports:
      - '127.0.0.1:6379:6379'
    healthcheck:
      test: ['CMD', 'redis-cli', 'ping']
      interval: 10s
      timeout: 3s
      retries: 5

  # flowcode Server (Go 二进制)
  server:
    image: flowcode/server:latest
    container_name: flowcode-server
    restart: unless-stopped
    depends_on:
      mysql:
        condition: service_healthy
      redis:
        condition: service_healthy
    environment:
      GIN_MODE: release
      DB_HOST: mysql
      DB_PORT: '3306'
      DB_USER: flowcode
      DB_PASSWORD: ${DB_PASSWORD:-flowcode_secret}
      DB_NAME: flowcode
      REDIS_URL: redis://redis:6379
      FLOWCODE_ENCRYPTION_KEY: ${ENCRYPTION_KEY:-change-me-to-a-random-32-char-string}
      JWT_SECRET: ${JWT_SECRET:-change-me-to-a-random-secret}
      SANDBOX_IMAGE: flowcode/sandbox:latest   # 沙盒容器镜像
      DOCKER_HOST: unix:///var/run/docker.sock  # Docker SDK 连接方式
      LOG_LEVEL: info
    volumes:
      - /var/run/docker.sock:/var/run/docker.sock  # Docker SDK
      - skills_data:/app/skills                     # Skills 文件存储
    ports:
      - '127.0.0.1:3000:3000'
    healthcheck:
      test: ['CMD', 'wget', '--quiet', '--tries=1', '--spider', 'http://localhost:3000/api/v1/health']
      interval: 30s
      timeout: 5s
      retries: 3

  # Nginx (反向代理 + 前端静态资源)
  nginx:
    image: nginx:alpine
    container_name: flowcode-nginx
    restart: unless-stopped
    ports:
      - '80:80'
      - '443:443'
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf
      - ./dist:/usr/share/nginx/html   # Vue 3 构建产物
      - ./ssl:/etc/nginx/ssl           # SSL 证书 (可选)
    depends_on:
      - server

volumes:
  mysql_data:
  redis_data:
  skills_data:
  # no sandbox_data — Docker manages containers & volumes internally

# ============================================
# 部署前需先构建沙盒镜像:
#   docker build -f Dockerfile.sandbox -t flowcode/sandbox:latest .
# ============================================
```

### Nginx 配置（生产环境推荐）

```nginx
# nginx.conf
worker_processes auto;
error_log /var/log/nginx/error.log warn;
pid /var/run/nginx.pid;

events {
    worker_connections 1024;
}

http {
    include       /etc/nginx/mime.types;
    default_type  application/octet-stream;

    log_format main '$remote_addr - $remote_user [$time_local] "$request" '
                    '$status $body_bytes_sent "$http_referer" '
                    '"$http_user_agent"';
    access_log /var/log/nginx/access.log main;

    sendfile        on;
    keepalive_timeout 65;
    gzip            on;
    gzip_types      text/plain text/css application/json application/javascript text/xml application/xml;

    # 后端 upstream — 多实例负载均衡
    upstream flowcode_backend {
        ip_hash;                         # WebSocket 会话保持
        server flowcode-1:3000;
        server flowcode-2:3000;
        server flowcode-3:3000;
        keepalive 32;
    }

    server {
        listen 80;
        server_name flowcode.example.com;

        # 安全头
        add_header X-Frame-Options "DENY" always;
        add_header X-Content-Type-Options "nosniff" always;
        add_header Referrer-Policy "strict-origin-when-cross-origin" always;

        # API 反向代理
        location /api/ {
            proxy_pass http://flowcode_backend;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
            proxy_set_header X-Forwarded-Proto $scheme;
        }

        # WebSocket 代理 (ip_hash 会话保持 + Redis Pub/Sub 跨实例)
        location /ws/ {
            proxy_pass http://flowcode_backend;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
            proxy_read_timeout 3600s;
            proxy_send_timeout 3600s;
        }

        # SDK 静态资源
        location /sdk/ {
            proxy_pass http://server:3000;
            proxy_http_version 1.1;
            proxy_set_header Host $host;
        }

        # 前端静态资源 (Vue 3 构建产物)
        location / {
            root   /usr/share/nginx/html;
            index  index.html;
            try_files $uri $uri/ /index.html;
        }
    }
}
```

## 10.3 环境变量清单

| 变量 | 必填 | 默认值 | 说明 |
|------|:---:|--------|------|
| `GIN_MODE` | 否 | release | Gin 运行模式 (debug/release/test) |
| `DB_HOST` | 是 | mysql | MySQL 主机地址 |
| `DB_PORT` | 是 | 3306 | MySQL 端口 |
| `DB_USER` | 是 | flowcode | MySQL 用户名 |
| `DB_PASSWORD` | 是 | - | MySQL 密码 |
| `DB_NAME` | 是 | flowcode | MySQL 数据库名 |
| `REDIS_URL` | 是 | redis://redis:6379 | Redis 连接串 |
| `FLOWCODE_ENCRYPTION_KEY` | **是** | - | 加密密钥 (32字符) |
| `JWT_SECRET` | **是** | - | JWT 签名密钥 |
| `SANDBOX_IMAGE` | 否 | flowcode/sandbox:latest | Docker 沙盒镜像 |
| `SANDBOX_MEMORY_MB` | 否 | 2048 | 每个沙盒容器内存限制 (MB) |
| `SANDBOX_CPUS` | 否 | 2 | 每个沙盒容器 CPU 核数 |
| `SANDBOX_DISK_MB` | 否 | 2048 | 每个沙盒容器磁盘限制 (MB) |
| `SANDBOX_TIMEOUT` | 否 | 600 | 沙盒执行超时 (秒) |
| `SANDBOX_NODES` | 否 | unix:///var/run/docker.sock | 分布式节点列表 (逗号分隔) |
| `DOCKER_HOST` | 否 | unix:///var/run/docker.sock | 本机 Docker SDK 连接地址 |
| `SANDBOX_CONFIG_FILE` | 否 | config/sandbox.yaml | 沙盒 YAML 配置文件路径 |
| `LOG_LEVEL` | 否 | info | 日志级别 (debug/info/warn/error) |
| `API_BASE_URL` | 否 | http://localhost:3000 | API 基础 URL |
| `SMTP_HOST` | 否 | - | 邮件服务器（通知用） |
| `SMTP_PORT` | 否 | 587 | 邮件端口 |
| `SMTP_USER` | 否 | - | 邮件账号 |
| `SMTP_PASS` | 否 | - | 邮件密码 |

## 10.4 一键启动

```bash
# 1. 克隆项目
git clone https://github.com/your-org/flowcode.git
cd flowcode

# 2. 构建 Vue 3 前端
cd web && npm install && npm run build && cd ..

# 3. 生成密钥
echo "ENCRYPTION_KEY=$(openssl rand -hex 16)" >> .env
echo "JWT_SECRET=$(openssl rand -hex 32)" >> .env
echo "DB_PASSWORD=$(openssl rand -hex 16)" >> .env

# 4. 启动所有服务
docker compose up -d

# 5. 数据库迁移（自动执行，Gorm AutoMigrate）
# 服务启动时自动通过 Gorm AutoMigrate 完成表结构同步

# 6. 访问
# Web UI:  http://localhost
# API:     http://localhost:3000
```

## 10.5 数据库迁移

使用 Gorm AutoMigrate 管理数据库 schema：

```go
// 服务启动时自动迁移
// internal/db/migrate.go

func AutoMigrate(db *gorm.DB) error {
    return db.AutoMigrate(
        &model.User{},
        &model.Organization{},
        &model.OrgMember{},
        &model.APIToken{},
        &model.Project{},
        &model.Issue{},
        &model.ExecutionLog{},
        &model.TestCase{},
        &model.Skill{},
        &model.MonitorEvent{},
        // ... 其他模型
    )
}
```

```bash
# Gorm AutoMigrate 在服务启动时自动执行，不需要手动运行迁移命令
# 生产环境如需手动控制，可通过环境变量关闭自动迁移：
# AUTO_MIGRATE=false docker compose up -d
```

## 10.6 备份策略

### 数据库备份

```bash
# 每日备份脚本 (cronjob)
#!/bin/bash
BACKUP_DIR="/backups/flowcode"
mkdir -p $BACKUP_DIR
docker compose exec -T mysql mysqldump -u flowcode -p$DB_PASSWORD flowcode | gzip > "$BACKUP_DIR/flowcode_$(date +%Y%m%d).sql.gz"
# 保留最近 30 天
find $BACKUP_DIR -mtime +30 -delete
```

### 完整备份

```bash
# 备份所有数据卷
docker run --rm \
  -v flowcode_mysql_data:/data \
  -v $(pwd)/backups:/backup \
  alpine tar czf /backup/mysql_$(date +%Y%m%d).tar.gz -C /data .
```

### 业务日志保留策略

| 日志类型 | 保留期 | 清理策略 | 说明 |
|---------|--------|---------|------|
| `login_logs` | 90 天 | 定时任务 DELETE `WHERE created_at < NOW() - INTERVAL 90 DAY` | 高频写入，仅保留近期用于异常检测 |
| `audit_logs` | 2 年 | 归档到冷存储后删除（可选） | 审计合规，不可随意删除 |
| `execution_logs` | 30 天 | 定时任务清理 | 执行 stdout/stderr 体积大 |
| 应用日志 (slog) | 30 天 | Docker logrotate | JSON 格式写入文件，按天轮转 |

> 审计日志如需更长期保留，建议接入 [Loki](https://grafana.com/oss/loki/) 或 ELK 进行集中采集与存储。

### 关键指标

| 指标 | 采集方式 | 告警阈值 |
|------|---------|---------|
| API 响应时间 | Gin middleware metrics | P95 > 2s |
| 任务队列积压 | Asynq GetPending | > 20 |
| 执行失败率 | ExecutionLog 统计 | > 30% |
| 磁盘使用 | 系统监控 | > 80% |

## 10.9 沙盒节点扩展

当 AI 执行任务量大时，可扩展专用沙盒节点。每个节点需运行 dockerd，flowcode Server 通过 Docker TCP socket 远程调度容器：

```yaml
# 沙盒节点 (独立服务器)
flowcode-sandbox-node:
  image: flowcode/server:latest
  command: ["./flowcode", "worker", "--queue=sandbox"]
  environment:
    REDIS_URL: redis://<server-ip>:6379
    DOCKER_HOST: unix:///var/run/docker.sock
  volumes:
    - /var/run/docker.sock:/var/run/docker.sock
  deploy:
    replicas: 3
```

```go
// 多节点 Docker 客户端池
func NewMultiNodeDockerClient() *clientPool {
    nodes := []string{
        os.Getenv("DOCKER_HOST"),           // 本机 "unix:///var/run/docker.sock"
        "tcp://sandbox-node1:2375",          // 节点 1
        "tcp://sandbox-node2:2375",          // 节点 2
    }
    return newClientPool(nodes)
}
```

容器通过 Asynq 从 Redis 队列拉取任务，执行完即销毁。

## 10.10 升级流程

```bash
# 1. 拉取新镜像
docker compose pull

# 2. 备份数据库
./scripts/backup.sh

# 3. 停止服务
docker compose down

# 4. 启动新版本（Gorm AutoMigrate 自动处理迁移）
docker compose up -d

# 5. 验证
curl http://localhost:3000/api/v1/health
```
