## 5.2 状态详解

| 状态 | 含义 | 可触发动作 |
|------|------|-----------|
| `draft` | 草稿 | enqueue / edit / delete |
| `queued` | 已进入执行队列 | cancel |
| `running` | AI 正在执行 | cancel / pause / ask (AI主动询问) |
| `paused` | 暂停中 | continue / cancel |
| `waiting_input` | AI 暂停等待用户回答 | answer / skip_question / cancel |
| `done` | 执行成功 | create_pr / retry / close |
| `failed` | 执行失败 | retry / edit / close |
| `cancelled` | 已取消 | retry / close |
| `pr_ready` | 代码就绪，待创建 PR | create_pr |
| `pr_created` | PR 已创建 | (Webhook 驱动后续状态) / reopen |
| `deployed` | 已部署 | close / reopen |
| `closed` | 已关闭 | reopen（重新打开后自动回到 `draft`） |

> **执行流程**：Issue 完成分析和计划后可直接入队执行（`draft` → `enqueue` → `running`）。执行计划可在入队前生成，也可在 Worker 真正开始执行前自动补齐。`close` 后 `reopen` 回到 `draft` 状态。
>
> **关闭/重开循环**：Issue 在 `done`、`failed`、`cancelled`、`deployed` 状态下均可 `close`。关闭后随时 `reopen`，重开后进入 `draft`，可重新生成计划并再次入队执行。

> looplab/fsm 是 Go 生态最成熟的轻量 FSM 库，提供声明式事件定义、回调钩子、Graphviz 可视化，零运维成本。

### 5.3.1 Issue 状态机定义

```go
// internal/fsm/issue_fsm.go

import (
    "context"
    "github.com/looplab/fsm"
    "gorm.io/gorm"
)

// IssueFSM 封装 Issue 状态机
type IssueFSM struct {
    *fsm.FSM
    db  *gorm.DB
    svc *deps  // 依赖注入容器 (见 5.3.4)
}

// NewIssueFSM 创建 Issue 状态机实例
// issue: 当前数据库中的 Issue 记录（用于获取当前状态）
func NewIssueFSM(issue *model.Issue, deps *FSMDependencies) *IssueFSM {
    f := &IssueFSM{db: deps.DB, svc: deps}

    f.FSM = fsm.NewFSM(
        issue.Status,  // 从 DB 恢复当前状态
        fsm.Events{
            //    事件名        源状态                          目标状态
            {Name: "cancel",    Src: []string{"draft", "queued", "running", "paused", "waiting_input"}, Dst: "cancelled"},

            // 执行链（完成分析和计划后可直接入队）
            {Name: "enqueue",   Src: []string{"draft"},         Dst: "queued"},
            {Name: "pickup",    Src: []string{"queued"},        Dst: "running"},
            {Name: "pause",     Src: []string{"running"},       Dst: "paused"},
            {Name: "resume",    Src: []string{"paused"},        Dst: "running"},
            {Name: "ask",           Src: []string{"running"},       Dst: "waiting_input"},   // AI 主动询问
            {Name: "answer",        Src: []string{"waiting_input"}, Dst: "running"},         // 用户回答后继续
            {Name: "skip_question", Src: []string{"waiting_input"}, Dst: "running"},         // 用户跳过问题
            {Name: "done",      Src: []string{"running"},       Dst: "done"},
            {Name: "fail",      Src: []string{"running", "pr_ready"}, Dst: "failed"},

            // 跳过执行（opencode 判定 needsCodeChange=false，直接标记完成）
            {Name: "skip",      Src: []string{"draft"},         Dst: "done"},

            // 重试 & PR
            {Name: "retry",     Src: []string{"failed", "cancelled", "done"}, Dst: "draft"},
            {Name: "mark_pr_ready", Src: []string{"done"},     Dst: "pr_ready"},
            {Name: "create_pr", Src: []string{"pr_ready"},     Dst: "pr_created"},
            {Name: "deploy",    Src: []string{"pr_created"},   Dst: "deployed"},

            // 关闭 ↔ 重新打开（可重复）
            {Name: "close",     Src: []string{"deployed", "done", "cancelled", "failed", "pr_created"}, Dst: "closed"},
            {Name: "reopen",    Src: []string{"cancelled", "closed", "done", "deployed", "pr_created"}, Dst: "draft"},
        },
        fsm.Callbacks{
            // ---- Issue 生命周期回调 ----
            "before_enqueue":          f.beforeEnqueue,
            "after_enqueue":           f.afterEnqueue,
            "before_pickup":           f.beforePickup,
            "after_pickup":            f.afterPickup,
            "after_done":              f.afterDone,
            "after_fail":              f.afterFail,
            "after_retry":             f.afterRetry,
            "after_cancel":            f.afterCancel,
            "before_create_pr":        f.beforeCreatePR,
            "after_create_pr":         f.afterCreatePR,
            "after_deploy":            f.afterDeploy,
            "after_close":             f.afterClose,
            "after_reopen":            f.afterReopen,
            "after_skip":              f.afterSkip,      // 跳过执行回调（发射 event）

            // ---- AI 主动询问回调 ----
            "before_ask":             f.beforeAsk,
            "after_ask":              f.afterAsk,
            "before_answer":          f.beforeAnswer,
            "after_answer":           f.afterAnswer,
            "after_skip_question":    f.afterSkipQuestion,

            // ---- 通用回调 ----
            "after_event":             f.afterAnyEvent,     // 所有事件完成后触发
            "leave_state":             f.leaveState,        // 离开任意状态
            "enter_state":             f.enterState,        // 进入任意状态
        },
    )
    return f
}
```

### 5.3.3 状态持久化与恢复

```go
// internal/fsm/persistence.go

// SaveState 将 fsm 当前状态写回 Issue 记录
func (f *IssueFSM) SaveState(ctx context.Context, issue *model.Issue) error {
    issue.Status = f.Current()
    return f.db.WithContext(ctx).Model(issue).Update("status", issue.Status).Error
}

// RestoreFSM 从数据库恢复 FSM 实例
func RestoreIssueFSM(ctx context.Context, db *gorm.DB, deps *FSMDependencies, issueID string) (*IssueFSM, *model.Issue, error) {
    var issue model.Issue
    if err := db.WithContext(ctx).First(&issue, "id = ?", issueID).Error; err != nil {
        return nil, nil, err
    }
    return NewIssueFSM(&issue, deps), &issue, nil
}

// Transition 执行状态转换 + 自动持久化
// 对外暴露的统一入口，业务层只需调用此方法
func (f *IssueFSM) Transition(ctx context.Context, issue *model.Issue, event string) error {
    if err := f.Event(ctx, event); err != nil {
        return fmt.Errorf("fsm: %w", err)
    }
    return f.SaveState(ctx, issue)
}
```

### 5.3.4 依赖注入 — 回调所需的全部依赖

```go
// internal/fsm/dependencies.go

// FSMDependencies 聚合 FSM 回调所需的所有外部依赖
// 通过此结构体注入，避免回调中直接引用全局变量
type FSMDependencies struct {
    DB            *gorm.DB
    TaskQueue     TaskQueue                       // 任务队列接口 (DB/Redis 双实现)
    SandboxCfg    *sandbox.SandboxGlobalConfig     // 沙盒全局配置 (含并发上限)
    SandboxManager *sandbox.SandboxManager     // Docker 沙盒管理 (问询答案写入等)
    HookRegistry  *service.HookRegistry
    GitService    *service.GitService
    AIToolService *service.AIToolService
}

// TaskQueue 任务队列接口 — 解耦具体队列实现
// 默认使用 DB 轮询 (dbq.go)，多实例场景可切换 Redis+Asynq
type TaskQueue interface {
    // Enqueue 投递任务到队列
    Enqueue(ctx context.Context, task *Task) error
    // Dequeue 从队列拉取待执行任务 (blocking / polling)
    Dequeue(ctx context.Context) (*Task, error)
    // Schedule 延迟投递 (scheduled_at)
    Schedule(ctx context.Context, task *Task, delay time.Duration) error
}

type Task struct {
    ID        string          `json:"id"`
    Type      string          `json:"type"`       // issue:execute / issue:analyze / ...
    Payload   json.RawMessage `json:"payload"`
    MaxRetry  int             `json:"max_retry"`
}
```

## 5.4 关键业务逻辑 — 回调实现

原有的审核、调度、重试等逻辑全部迁移到 fsm 回调中，保持业务逻辑与状态转换解耦。

### 5.4.1 执行调度 (draft → queued → running)

```go
// beforeEnqueue — 入队前置检查
func (f *IssueFSM) beforeEnqueue(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)

    // 校验前置分析
    if !issue.AnalysisCompleted {
        e.Cancel(fmt.Errorf("issue analysis not completed"))
        return
    }

    // 校验 AI 工具配置
    tool, err := f.svc.DB.GetAIToolConfig(ctx, issue.ProjectID, issue.AITool)
    if err != nil || !tool.Enabled {
        e.Cancel(fmt.Errorf("AI tool not configured: %s", issue.AITool))
        return
    }
    e.Args = append(e.Args, tool)
}

// afterEnqueue — 投递异步任务
func (f *IssueFSM) afterEnqueue(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)

    // 创建执行日志
    log := &model.ExecutionLog{
        IssueID: issue.ID,
        Status:  "queued",
        Attempt: issue.Attempt + 1,
    }
    f.db.Create(log)

    // 投递任务队列
    task := &Task{
        ID:       uuid.New().String(),
        Type:     "issue:execute",
        Payload:  marshal(ExecutePayload{
            IssueID: issue.ID,
            LogID:   log.ID,
        }),
        MaxRetry: 0,
    }
    f.svc.TaskQueue.Enqueue(ctx, task)

    f.svc.HookRegistry.Emit(ctx, "issue:queued", issue)
}

// beforePickup — Worker 拾取任务时触发，检查两级并发限制
func (f *IssueFSM) beforePickup(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)
    project, _ := f.svc.DB.GetProject(ctx, issue.ProjectID)

    // ── 第一级：全局并行上限 (跨所有项目) ──
    globalMax := f.svc.SandboxCfg.MaxConcurrentExecutions
    if globalMax <= 0 {
        globalMax = 10 // 默认值
    }
    var globalRunning int64
    f.db.Model(&model.Issue{}).
        Where("status = ?", "running").
        Count(&globalRunning)
    if int(globalRunning) >= globalMax {
        e.Cancel(fmt.Errorf("global concurrency limit reached: %d/%d", globalRunning, globalMax))
        return
    }

    // ── 第二级：项目并行上限 ──
    maxConcurrent := project.Settings.MaxConcurrentExecutions
    if maxConcurrent <= 0 {
        maxConcurrent = 3 // 默认值
    }
    var runningCount int64
    f.db.Model(&model.Issue{}).
        Where("project_id = ? AND status = ?", issue.ProjectID, "running").
        Count(&runningCount)
    if int(runningCount) >= maxConcurrent {
        e.Cancel(fmt.Errorf("project concurrency limit reached: %d/%d", runningCount, maxConcurrent))
        return
    }

    f.svc.HookRegistry.Emit(ctx, HookIssueExecutionStart, e.Args[0])
}
```

Worker 侧的状态驱动 (Queue Worker — DB/Asynq 共用 handler)：

```go
// internal/queue/handler.go

func ExecuteIssueHandler(ctx context.Context, t *Task) error {
    var payload ExecutePayload
    json.Unmarshal(t.Payload, &payload)

    // 1. 恢复状态机
    fsm, issue, _ := RestoreIssueFSM(ctx, db, deps, payload.IssueID)

    // 2. 前置检查: opencode 分析结果
    if issue.AnalysisCompleted && !issue.NeedsCodeChange {
        // 无需代码变更（纯文档/配置/沟通类 Issue），直接标记完成
        slog.Info("Issue needs no code change, auto-completing",
            "issue", issue.ID, "analysis", issue.AnalysisReason)
        fsm.Transition(ctx, issue, "skip")
        return nil
    }

    // 3. 生成执行计划（若尚未生成）
    if !issue.PlanCompleted {
        plan, err := deps.AIToolService.Plan(ctx, issue)
        if err != nil {
            return err
        }
        issue.PlanSummary = plan.Summary
        issue.PlanDetail = plan.Detail
        issue.PlanCompleted = true
        issue.PlanGeneratedAt = time.Now()
        db.Save(issue)
        deps.HookRegistry.Emit(ctx, HookIssuePlanned, issue)
    }

    // 4. 入队 → 执行中
    fsm.Transition(ctx, issue, "pickup")

    // 5. 调用 AI 工具
    result, err := deps.AIToolService.Execute(ctx, issue)

    // 6. 根据结果驱动状态
    if err != nil {
        fsm.Transition(ctx, issue, "fail")
    } else {
        fsm.Transition(ctx, issue, "done")
    }
    return nil
}
```

**DB 队列模式 Worker 轮询**（`FLOWCODE_QUEUE_DRIVER=database`）：

```go
// internal/queue/dbq.go

func (w *DBQueueWorker) Run(ctx context.Context) {
    ticker := time.NewTicker(w.pollInterval) // 默认 2s
    defer ticker.Stop()

    for {
        select {
        case <-ticker.C:
            task, err := w.dequeue(ctx)
            if err != nil || task == nil {
                continue // 无任务，等待下一轮
            }
            // 调用对应的 handler（与 Asynq 模式共用同一 handler）
            if task.Type == "issue:execute" {
                w.executeHandler(ctx, task)
            }
        case <-ctx.Done():
            return
        }
    }
}

func (w *DBQueueWorker) dequeue(ctx context.Context) (*Task, error) {
    var t model.TaskQueue
    err := w.db.WithContext(ctx).
        Where("status = ? AND (scheduled_at IS NULL OR scheduled_at <= ?)", "pending", time.Now()).
        Order("priority ASC, created_at ASC").
        First(&t).Error
    if errors.Is(err, gorm.ErrRecordNotFound) {
        return nil, nil
    }
    // 原子抢占: UPDATE status='running', started_at=NOW()
    w.db.Model(&t).Updates(map[string]any{"status": "running", "started_at": time.Now()})
    return t.ToTask(), err
}
```

```go
// POST /api/v1/issues/:id/continue
func (h *IssueHandler) Continue(c *gin.Context) {
    fsm, issue, _ := RestoreIssueFSM(c.Request.Context(), h.db, h.deps, c.Param("id"))

    // done / failed → 重新入队
    if fsm.InState("done", "failed") {
        fsm.Transition(ctx, issue, "retry")
        fsm.Transition(ctx, issue, "enqueue")
    }
    // paused → 恢复执行
    if fsm.InState("paused") {
        fsm.Transition(ctx, issue, "resume")
    }
    // 保留原 gitBranch，AI 在原分支基础上继续
    c.JSON(200, issue)
}
```

```go
func (f *IssueFSM) beforeCreatePR(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)

    // 前置: 必须有 Git 凭证
    cred, err := f.svc.DB.GetGitCredential(ctx, issue.ProjectID)
    if err != nil {
        e.Cancel(fmt.Errorf("git credential not found"))
        return
    }

    // 提交变更
    commitSha, _ := f.svc.GitService.CommitChanges(ctx, issue.GitBranch, issue)
    issue.GitCommitSha = commitSha
    e.Args = append(e.Args, cred)
}

func (f *IssueFSM) afterCreatePR(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)

    // 创建 PR
    pr, _ := f.svc.GitService.CreatePR(ctx, issue)
    issue.GitPRUrl = pr.URL
    f.db.Save(issue)

    f.svc.HookRegistry.Emit(ctx, "issue:pr:created", issue, pr)
}

// 简化端到端流程
func (f *IssueFSM) AutoCreatePRPipeline(ctx context.Context, issue *model.Issue) error {
    if err := f.Transition(ctx, issue, "mark_pr_ready"); err != nil {
        return err
    }
    return f.Transition(ctx, issue, "create_pr")
}
```

### 5.4.4 失败重试与回退

```go
// internal/fsm/issue_callbacks.go

func (f *IssueFSM) afterFail(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)
    project, _ := f.svc.DB.GetProject(ctx, issue.ProjectID)

    maxRetries := project.Settings.MaxRetries  // 默认 3
    if issue.Attempt < maxRetries {
        // 自动降级重试
        f.Event(ctx, "retry")
        f.Event(ctx, "enqueue")
        f.svc.HookRegistry.Emit(ctx, "issue:execution:retry", issue, issue.Attempt)
        return
    }

    // 超过最大重试 → 保持在 failed，通知管理员
    f.svc.HookRegistry.Emit(ctx, "issue:execution:failed", issue)
}
```

### 5.4.5 AI 主动询问 (running → waiting_input → running)

```go
// beforeAsk — 检查项目是否启用 AI 询问功能
func (f *IssueFSM) beforeAsk(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)
    project, _ := f.svc.DB.GetProject(ctx, issue.ProjectID)

    // 检查是否启用 AI 询问

    }
}

// afterAsk — AI 发起询问后，创建 AIQuestion 记录并通知用户
func (f *IssueFSM) afterAsk(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)
    questionData := e.Args[1].(*AIQuestionInput)  // {Question, Context, Options}

    // 创建问询记录
    question := &model.AIQuestion{
        ID:             uuid.New().String(),
        IssueID:        issue.ID,
        ExecutionLogID: issue.LatestExecutionLogID,
        Question:       questionData.Question,
        Context:        questionData.Context,
        Options:        questionData.Options,  // JSON array
        Status:         "pending",
        AskedAt:        time.Now(),
    }
    f.db.Create(question)

    // 更新 Issue 待回答问题计数
    f.db.Model(issue).Update("pending_question_count", gorm.Expr("pending_question_count + 1"))

    // 发射 Hook 事件
    f.svc.HookRegistry.Emit(ctx, HookIssueQuestionAsked, issue, question)

    // 启动超时计时器（若配置了超时）
    project, _ := f.svc.DB.GetProject(ctx, issue.ProjectID)
    if timeout := project.Settings.AIQuestionTimeout; timeout > 0 {
        go f.scheduleQuestionTimeout(ctx, question.ID, issue.ID, timeout)
    }
}

// beforeAnswer — 验证答案
func (f *IssueFSM) beforeAnswer(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)

    // 检查是否有待回答的问题
    var count int64
    f.db.Model(&model.AIQuestion{}).
        Where("issue_id = ? AND status = ?", issue.ID, "pending").
        Count(&count)
    if count == 0 {
        e.Cancel(fmt.Errorf("no pending questions for this issue"))
    }
}

// afterAnswer — 用户回答后，将答案写入沙盒文件并继续 AI 执行
func (f *IssueFSM) afterAnswer(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)
    questionID := e.Args[1].(string)
    answer := e.Args[2].(string)

    // 更新问询记录
    f.db.Model(&model.AIQuestion{}).Where("id = ?", questionID).Updates(map[string]any{
        "status":     "answered",
        "answer":     answer,
        "answered_by": ctx.Value("userID"),
        "answered_at": time.Now(),
    })

    // 更新 Issue 待回答问题计数
    f.db.Model(issue).Update("pending_question_count", gorm.Expr("GREATEST(pending_question_count - 1, 0)"))

    // 将答案写入沙盒文件
    answerContent := fmt.Sprintf("## User Answer\n\n**Question**: %s\n\n**Answer**: %s\n",
        e.Args[3].(string), answer)
    f.svc.SandboxManager.WriteAnswer(ctx, issue.ID, answerContent)

    // 发射 Hook 事件
    f.svc.HookRegistry.Emit(ctx, HookIssueQuestionAnswered, issue, questionID, answer)
}

// afterSkipQuestion — 用户跳过问题，AI 自行决策继续
func (f *IssueFSM) afterSkipQuestion(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)
    questionID := e.Args[1].(string)

    // 标记问题为已跳过
    f.db.Model(&model.AIQuestion{}).Where("id = ?", questionID).Updates(map[string]any{
        "status":     "skipped",
        "answered_at": time.Now(),
    })

    // 更新 Issue 待回答问题计数
    f.db.Model(issue).Update("pending_question_count", gorm.Expr("GREATEST(pending_question_count - 1, 0)"))

    // 写入跳过标记
    f.svc.SandboxManager.WriteAnswer(ctx, issue.ID,
        "## User Decision\n\n**Skipped** — proceed with your best judgment.\n")
}

// scheduleQuestionTimeout — 超时自动跳过问题
func (f *IssueFSM) scheduleQuestionTimeout(ctx context.Context, questionID, issueID string, timeoutSec int) {
    select {
    case <-time.After(time.Duration(timeoutSec) * time.Second):
        fsm, issue, _ := RestoreIssueFSM(ctx, f.db, f.svc, issueID)
        if issue.Status == "waiting_input" {
            // 自动跳过超时问题
            fsm.Transition(ctx, issue, "skip_question")
        }
    case <-ctx.Done():
        return
    }
}
```

## 5.5 工作流 Hook 点 (集成到 FSM 回调)

Hook 事件不再单独维护列表，而是直接在 fsm 回调中 emit。事件常量保留用于外部订阅：

## 5.7 历史记录自动采集

历史记录通过 **fsm 的 `after_event` / `enter_state` 回调** 统一采集，替代原来分散在多个 Gorm Hook 中的逻辑。

### 5.7.1 通用历史记录回调

```go
// internal/fsm/history_hook.go

// afterAnyEvent — 所有 fsm 事件结束后自动记录历史
func (f *IssueFSM) afterAnyEvent(ctx context.Context, e *fsm.Event) {
    // 记录状态变更
    _ = f.db.WithContext(ctx).Create(&model.HistoryRecord{
        TargetType: "issue",
        TargetId:   e.Args[0].(*model.Issue).ID,
        Action:     "status_changed",
        FieldName:  "status",
        OldValue:   mustJSON(map[string]string{"status": e.Src}),
        NewValue:   mustJSON(map[string]string{"status": e.Dst}),
        ChangedBy:  ctx.Value("userID").(string),
    }).Error
}

// enterState — 进入特定状态时的附加记录
func (f *IssueFSM) enterState(ctx context.Context, e *fsm.Event) {
    issue := e.Args[0].(*model.Issue)
    // 例如进入 deployed 状态额外记录
    if e.Dst == "deployed" {
        f.svc.HookRegistry.Emit(ctx, HookIssueDeployed, issue)
    }
}

// leaveState — 离开状态时的清理逻辑
func (f *IssueFSM) leaveState(ctx context.Context, e *fsm.Event) {
    // 离开 running 状态时，关闭 WebSocket 连接等
    if e.Src == "running" {
        f.svc.WsHub.CloseConn(e.Args[0].(*model.Issue).ID)
    }
}
```

### 5.7.2 字段级变更仍然使用 Gorm Hook

状态变更由 fsm `after_event` 处理；标题/描述等字段的修改仍然用 Gorm Hook（见 [03-数据模型.md 5.8 原实现]）。两者互不冲突：

```
[Issue 状态变更]  → fsm after_event    → HistoryRecord (status_changed)
[Issue 字段修改]  → Gorm AfterUpdate   → HistoryRecord (updated, per field)
[Issue 创建]      → Gorm AfterCreate   → HistoryRecord (created)
```

### 5.7.3 注册

Gorm AutoMigrate 时自动绑定 Gorm Hook；fsm 回调在 `NewIssueFSM()` 初始化时注册。

## 5.8 Webhook 驱动的外部状态变更

某些状态转换不由用户手动触发，而是由外部事件（Git Webhook）驱动：

### Git 平台 Webhook → FSM

```go
// internal/handler/git_webhook.go

func (h *GitHandler) HandleWebhook(provider ProviderType) gin.HandlerFunc {
    return func(c *gin.Context) {
        // ... 解析 PR/MR Webhook 事件（见 08-Git集成方案 8.11）

        ctx := c.Request.Context()

        // 恢复 FSM
        fsm, issue, _ := RestoreIssueFSM(ctx, h.db, h.deps, issueID)

        if action == "closed" && merged {
            // PR/MR merged → deployed → closed
            fsm.Transition(ctx, issue, "deploy")
            fsm.Transition(ctx, issue, "close")
        } else if action == "closed" && !merged {
            // PR closed but not merged → reopen (可重新执行)
            fsm.Transition(ctx, issue, "reopen")
        }
    }
}
```

### Webhook 防重入

```go
func (f *IssueFSM) Transition(ctx context.Context, issue *model.Issue, event string) error {
    // 乐观锁: 仅当 DB 中的状态与 fsm 当前状态一致时才执行
    result := f.db.WithContext(ctx).
        Model(issue).
        Where("status = ?", f.Current()).
        Update("status", fsm.StateTo(e))  // 预判目标状态

    if result.RowsAffected == 0 {
        return fmt.Errorf("concurrent state change detected, retry")
    }

    // 更新成功，本地 fsm 同步
    return f.Event(ctx, event)
}
```

### 事件 → FSM 映射表

| 外部事件源 | 事件 | FSM 事件 |
|-----------|------|----------|
| Git 平台 Webhook | PR/MR merged | `deploy` → `close` |
| Git 平台 Webhook | PR/MR closed (no merge) | `reopen` |
| Queue Worker | AI execution done | `done` |
| Queue Worker | AI execution failed | `fail` |
| CLI | `flowcode issue execute` | `enqueue` |
