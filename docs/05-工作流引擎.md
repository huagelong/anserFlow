# 05 - 工作流引擎

## 5.1 Issue 生命周期状态机

## 5.2 状态详解

| 状态 | 含义 | 可触发动作 |
|------|------|-----------|
| `draft` | 草稿，待审核 | approve / reject / edit / delete |
| `approved` | 审核通过，等待执行 | execute / edit / reject |
| `rejected` | 审核驳回 | reopen / delete |
| `queued` | 已进入执行队列 | cancel |
| `running` | AI 正在执行 | cancel / pause |
| `paused` | 暂停中 | continue / cancel |
| `done` | 执行成功 | create_pr / retry |
| `failed` | 执行失败 | retry / edit / reject |
| `cancelled` | 已取消 | retry / reopen |
| `pr_ready` | 代码就绪，待创建 PR | create_pr / retry |
| `pr_created` | PR 已创建 | (由 Webhook 驱动后续状态) |
| `deployed` | 已部署 | close / reopen |
| `closed` | 已关闭（终态） | reopen |
| `reopened` | 已重新打开 | → 回到 approved |

## 5.3 状态转换守卫

```go
// internal/service/issue_state.go

// 状态转换规则（状态机核心）
var transitions = map[string][]string{
    "draft":      {"approved", "rejected", "cancelled"},
    "approved":   {"queued"},
    "rejected":   {"draft"},
    "queued":     {"running", "cancelled"},
    "running":    {"done", "failed", "cancelled", "paused"},
    "paused":     {"running", "cancelled"},
    "done":       {"pr_ready"},
    "failed":     {"approved", "cancelled", "draft"},
    "cancelled":  {"draft"},
    "pr_ready":   {"pr_created", "failed"},
    "pr_created": {"deployed", "reopened"},
    "deployed":   {"closed"},
    "closed":     {"reopened"},
    "reopened":   {"approved"},
}

func CanTransition(from, to string) bool {
    allowed, ok := transitions[from]
    if !ok {
        return false
    }
    for _, s := range allowed {
        if s == to {
            return true
        }
    }
    return false
}
```

## 5.4 关键业务逻辑

### 5.4.1 审核流程 (draft → approved / rejected)

```
trigger: POST /api/issues/:id/approve 或 /reject

preconditions:
  - issue.status === 'draft'
  - 操作者 role === 'admin'

actions:
  approve:
    - status → 'approved'
    - 如果 project.settings.autoExecute === true → 自动进入 queued
  reject:
    - status → 'rejected'
    - 记录驳回原因到 issue.aiConfig.rejectReason
```

### 5.4.2 执行调度 (approved → queued → running)

```
trigger: POST /api/issues/:id/execute (或自动触发)

preconditions:
  - issue.status === 'approved'
  - issue.aiTool 对应的 AIToolConfig.enabled === true

actions:
  1. 校验 AI 工具配置
  2. status → 'queued'
  3. 创建 ExecutionLog (status='queued', attempt++)
  4. 投递 Asynq 任务
  5. Worker 拾取任务:
     a. status → 'running'
     b. 通过 AIToolAdapter 调用 AI 工具
     c. 实时推送 stdout → WebSocket
     d. 执行完成: status → 'done'
     e. 执行失败: status → 'failed', 记录 stderr
```

### 5.4.3 继续编辑 (vibe coding 续跑)

```
trigger: POST /api/issues/:id/continue

preconditions:
  - issue.status ∈ ['done', 'failed', 'paused']
  - issue.gitBranch 存在（有之前的代码上下文）

actions:
  1. 保留当前 gitBranch
  2. 创建新的 ExecutionLog (attempt++)
  3. 用户可补充新的自然语言指令
  4. AI 在原分支基础上继续修改
  5. WebSocket 推送新日志
```

### 5.4.4 PR 创建 (done → pr_ready → pr_created)

```
trigger: POST /api/projects/:pid/git/pr (或自动触发)

preconditions:
  - issue.status === 'done' 或 'pr_ready'
  - project.gitProvider 已配置
  - GitCredential 有效

actions:
  1. status → 'pr_ready'
  2. GitProvider.commitChanges()
  3. GitProvider.createPullRequest({
       title: issue.title,
       body: prTemplate(issue),
       base: project.defaultBranch,
       head: issue.gitBranch
     })
  4. 保存 prUrl 到 issue.gitPRUrl
  5. status → 'pr_created'
```

### 5.4.5 失败重试与回退

```
重试策略 (Project.settings.maxRetries):

attempt 1: 失败 → 自动重试（同参数）
attempt 2: 失败 → 自动重试（降级 temperature，添加更详细的 prompt）
attempt 3: 失败 → status → 'failed'，通知管理员

人工介入方式:
  - POST /api/issues/:id/continue  → 补充指令后继续
  - PUT /api/issues/:id (修改描述) → 再 retry
  - PUT /api/issues/:id/status → 'draft' → 回退到编辑
```

## 5.7 TestCase 审核工作流

TestCase 必须先经过审核，才能调用 AI 工具生成测试代码：

```
                        ┌──────────┐
                        │  draft   │  ← 用户创建测试描述
                        └────┬─────┘
                             │ user 提交审核
                             ▼
                       ┌──────────────┐
                       │ pending_review│
                       └──────┬───────┘
                              │ admin 审核
              ┌───────────────┼───────────────┐
              │ approve       │               │ reject
              ▼               │               ▼
        ┌──────────┐         │         ┌──────────┐
        │ approved │         │         │ rejected │ ──→ 用户修改后重新提交
        └────┬─────┘         │         └──────────┘
             │               │
             ▼               │
       ┌────────────┐        │
       │ generating │  ← AI   │
       └─────┬──────┘        │
             │               │
             ▼               │
       ┌────────────┐        │
       │ generated  │        │

             │               │
```

### TestCase 状态转换守卫

```go
// internal/service/test_case_state.go

var testCaseTransitions = map[string][]string{
    "draft":          {"pending_review"},
    "pending_review": {"approved", "rejected"},
    "rejected":       {"draft"},
    "approved":       {"generating"},
    "generating":     {"generated", "rejected"},
    "generated":      {"passed", "failed"},
    "failed":         {"draft", "approved"}, // 修正后重新生成
    "passed":         {},                      // 终态

```

### 审核通过后的自动流程

```
trigger: POST /api/v1/test-cases/:id/approve

preconditions:
  - testCase.reviewStatus === 'pending_review'
  - 操作者 role === 'admin'

post-actions:
  1. reviewStatus → 'approved'
  2. 自动触发 AI 生成: POST /api/v1/test-cases/:id/regenerate
  3. AI 根据 testCase.description + issue.codeContext 生成测试代码
  4. 保存生成的 testCode
  5. reviewStatus → 'generated'
  6. 自动运行测试
  7. 记录执行结果到 testCase.result
  8. status → 'passed' | 'failed'
```

## 5.8 历史记录自动采集

所有核心实体的变更通过 Gorm Hook 自动记录到 `history_records` 表：

```go
// internal/model/history.go

// Gorm 回调 — 创建记录
func (i *Issue) AfterCreate(tx *gorm.DB) error {
    return tx.Create(&HistoryRecord{
        TargetType: "issue",
        TargetId:   i.ID,
        Action:     "created",
        FieldName:  "*",
        NewValue:   mustJSON(i),
        ChangedBy:  getCurrentUserID(tx),
    }).Error
}

// Gorm 回调 — 更新记录 (仅记录变更字段)
func (i *Issue) AfterUpdate(tx *gorm.DB) error {
    changes := tx.Statement.Changed()
    for _, field := range changes {
        oldVal := mustJSON(tx.Statement.ReflectValue.FieldByName(field).Interface())
        newVal := mustJSON(reflect.ValueOf(i).Elem().FieldByName(field).Interface())

        action := "updated"
        if field == "Status" {
            action = "status_changed"
        }

        tx.Create(&HistoryRecord{
            TargetType: "issue",
            TargetId:   i.ID,
            Action:     action,
            FieldName:  toSnakeCase(field),
            OldValue:   oldVal,
            NewValue:   newVal,
            ChangedBy:  getCurrentUserID(tx),
        })
    }
    return nil
}

// 同理适用于 Project / Requirement / TestCase
```

### 历史记录 Hook 注册

在 Gorm AutoMigrate 时自动绑定：

```go
func AutoMigrate(db *gorm.DB) error {
    // 注册所有需要历史记录的模型
    models := []any{
        &Project{},
        &Requirement{},
        &Issue{},
        &TestCase{},
        &HistoryRecord{},
    }
    return db.AutoMigrate(models...)
}
```
