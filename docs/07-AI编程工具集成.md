# 07 - AI 编程工具集成

## 7.1 概述

flowcode 通过标准 `AIToolAdapter` 接口集成主流 AI 编程工具。所有工具在 **Docker 容器沙盒**中执行，确保 AI 生成的任意代码（包括 `curl | sh`、`npm install`、`rm -rf`）完全与宿主机隔离。

> **决策**：统一使用 Docker，不引入 Bubblewrap/nsjail 等多方案。Docker 在分布式部署、资源限制、镜像缓存、日志驱动方面生态最成熟。

### 7.1b 通信机制

flowcode 与 AI 工具之间不依赖任何网络协议或 RPC，通过 **文件注入 + 子进程 CLI + 管道捕获** 三层操作系统级通道通讯：

```
┌──────────────────────────────────────────────────┐
│              flowcode Server (Go)                │
│                                                  │
│  ① 写入 .flowcode_prompt.md                      │
│  ② exec.CommandContext("opencode", "run", ...)   │
│  ③ stdoutPipe / stderrPipe 实时捕获              │
│  ④ cmd.Wait() → exitCode 判定结果               │
└────────────┬─────────────────────────────────────┘
             │ Docker SDK
┌────────────┴─────────────────────────────────────┐
│            Docker 容器（沙盒）                     │
│                                                  │
│  /workspace/                                     │
│  ├── .flowcode_prompt.md   ← ① Prompt 注入       │
│  └── repo/                 ← AI 工具读写代码      │
│                                                  │
│  opencode CLI 子进程                             │
│  ├── stdin   (不使用)                            │
│  ├── stdout  ──→ ③ pipe → WebSocket 推前端       │
│  └── stderr  ──→ ③ pipe                         │
└──────────────────────────────────────────────────┘
```

| 方向 | 通道 | 内容 | 说明 |
|------|------|------|------|
| flowcode → AI 工具 | **文件系统** | `.flowcode_prompt.md` | Issue 标题+描述+分析摘要+执行计划+Skills 指令，写入容器 `/workspace` |
| flowcode → AI 工具 | **环境变量** | `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` 等 | 通过 `cmd.Env` 注入，不落地文件 |
| flowcode 启动 AI | **子进程** | `exec.CommandContext` | 在沙盒容器内以 CLI 方式启动 AI 工具 |
| AI 工具 → flowcode | **stdout pipe** | 实时执行日志（JSON Lines） | 流式捕获 → `ctx.OnLog()` → WebSocket 推送前端 |
| AI 工具 → flowcode | **stderr pipe** | 警告/错误信息 | 流式捕获，level=warn |
| AI 工具 → flowcode | **退出码** | 0=成功, 非0=失败 | `cmd.Wait()` 获取 |
| AI 工具 ↔ 代码仓库 | **文件系统** | `/workspace/repo` | AI 工具通过标准文件读写操作源代码 |

**分析/分类场景**（无需代码仓库）：

```go
// 直接通过 --input 传 prompt，stdout 取 JSON 结果
cmd := exec.CommandContext(ctx, "opencode", "run",
    "--input", prompt,
    "--model", "claude-haiku",
    "--max-turns", "1",
    "--output-format", "json",
)
output, _ := cmd.CombinedOutput()
// 解析 output 中的 JSON
```

> **为什么不走网络协议？** AI 工具均为 CLI 程序，子进程是最直接的调用方式。管道捕获 stdout/stderr 天然支持流式输出，无需实现 WebSocket client 端或 gRPC stream。Docker 容器提供进程级隔离和资源限制。

### 7.1c 触发 opencode 开始开发

有三种方式触发 opencode 执行开发任务：

#### 方式 1：CLI（推荐）

```bash
# 一条命令：创建 Issue + 触发 opencode 执行
flowcode issue create "修复登录页面样式错乱" --priority p0 --tool opencode

# 指定 Skills + AI 配置
flowcode issue create "实现用户认证" -p p0 --tool claude-code \
  --skill auth,testing \
  --config '{"model":"claude-sonnet-4-20250514","maxTurns":60}'

# 实时查看执行日志
flowcode issue watch abc123
```

CLI 自动完成 **创建** 并触发前置分析；审批通过后可先生成执行计划，真正执行前若缺失则由 Queue Worker 自动补齐。

#### 方式 2：Web 控制台

```

  ├── 1. 点击「新建 Issue」 → 填写标题/描述 → 选择 AI 工具 → 提交

  │
  ├── 2. 管理员点击「审批通过」
  │      → Issue 状态: approved

  └── 3. 若项目开启 autoExecute → 自动入队执行
         否则 → 管理员点击「执行」→ 手动入队

              → 底部面板实时显示执行日志流
```


#### 底层执行链路

```
Issue approve / plan / execute
    │
    ▼
afterEnqueue() -> 投递 "issue:execute" 任务到队列
    │
    ▼
Worker pickup → RestoreIssueFSM() → Transition("pickup")
    │
    ▼
ExecuteIssueHandler()
    │
    ├── ① 前置分析检查（必要时补做 analyze）
    │
    ├── ② needsCodeChange 检查（false → skip，跳过执行）
    │
    ├── ③ 生成执行计划（summary + detail + test strategy）
    │
    ├── ④ SandboxManager.CreateContainer()
    │      └── 拉取 flowcode/sandbox:latest 镜像
    │      └── 挂载 volume flowcode-sandbox-{issueId} → /workspace
    │
    ├── ⑤ git clone {project.gitRepoUrl} → /workspace/repo
    │
    ├── ⑥ 注入 Skills → /workspace/.flowcode_prompt.md
    │      └── 内容: Issue 标题 + 描述 + 分析摘要 + 执行计划 + 绑定 Skill 指令
    │
    ├── ⑦ exec.CommandContext("opencode", "run",
    │      "--prompt-file", "/workspace/.flowcode_prompt.md",
    │      "--workspace", "/workspace/repo",
    │      "--model", "claude-sonnet-4-20250514",
    │      "--max-turns", "50")
    │
    ├── ⑥ stdout/stderr pipe → ctx.OnLog() → WebSocket → 前端实时日志
    │
    └── ⑦ cmd.Wait() → exitCode
           ├── 0 → Transition("done")
           │      → git add -A → git commit → git push
           │      → create PR → Transition("pr_created")
           └── ≠0 → Transition("fail") → retry 或通知管理员
```

> **autoExecute 开关**：在项目设置中开启后，审批通过即自动入队，无需手动点击「执行」。适合个人项目或信任的自动化场景。

### 7.1d AI 主动询问机制

AI 在执行编码任务时，如遇到歧义点（技术方案选择、缺失配置、边界条件确认等），可通过 stdout 输出特殊标记主动暂停并向用户提问。用户回答后 AI 继续执行，形成人机协同决策闭环。

#### 交互流程

```
AI 正在执行 (running)
    │
    │ AI 在 stdout 输出 [FLOWCODE_Q]
    ▼
flowcode 检测到问题标记
    │
    ├── ① 暂停 AI 子进程（SIGSTOP / keep container alive）
    ├── ② FSM: running → (ask) → waiting_input
    ├── ③ 创建 AIQuestion 记录
    ├── ④ 发射 Hook → WebSocket 通知前端 / Webhook 通知外部
    │
    ▼
用户查看问题
    │
    ├── 📝 回答 → 写入 /workspace/.flowcode_answer.md
    │              → FSM: waiting_input → (answer) → running
    │              → AI 子进程 continue (SIGCONT)
    │
    └── ⏭️ 跳过 → 写入跳过标记
                   → FSM: waiting_input → (skip_question) → running
                   → AI 子进程 continue (自行决策)
```

#### 问题标记格式

AI 工具在 stdout 输出的每行日志中，以特殊前缀 `[FLOWCODE_Q]` 标记问询内容。flowcode 的 stdout 解析器检测到该标记后触发问询流程：

```json
[FLOWCODE_Q]{
  "question": "检测到项目同时存在 src/components/ 和 components/ 目录，应将新组件放在哪个目录下？",
  "context": "- 当前文件: App.tsx\n- src/components/ 包含 12 个组件\n- components/ 包含 3 个组件（较旧）",
  "options": ["src/components/ (推荐)", "components/", "统一到 src/components/ 并迁移旧组件"]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|:---:|------|
| question | string | ✅ | 提问内容，面向人类用户的可读问题 |
| context | string | - | 上下文信息，Markdown 格式，帮助用户理解背景 |
| options | string[] | - | 可选答案列表，前端以按钮/下拉形式呈现 |

#### Prompt 模板注入

AI 执行前，flowcode 在 `.flowcode_prompt.md` 中注入问询规则指令，告知 AI 工具何时以及如何发起问询：

```markdown
## 问询规则 (Inquiry Rules)

如果你在执行任务时遇到以下情况，请通过 stdout 输出 `[FLOWCODE_Q]` 标记来向用户提问：

1. **方案选择**：有多种技术实现方案且各有利弊，需要人工决策
2. **缺失配置**：需要数据库连接串、API Key、环境变量等敏感信息
3. **边界确认**：需求描述不明确，可能影响核心逻辑设计
4. **冲突处理**：代码库存在冲突/不兼容，需要确认取舍

**格式要求**：
- 输出必须是单行 JSON，以 `[FLOWCODE_Q]` 开头
- 问询后暂停代码修改，等待用户响应
- 用户回答将写入 `/workspace/.flowcode_answer.md`
- 收到回答或跳过标记后继续执行

**不要问询的情况**：
- 简单的是/否问题（自行判断即可）
- 已经在 Skills / 执行计划中明确说明的内容
- 纯粹的代码风格/命名问题
```

#### stdout 解析器实现

```go
// internal/service/sandbox/question_detector.go

type QuestionDetector struct {
    onQuestion func(parsed *AIQuestionInput) error
}

var questionRe = regexp.MustCompile(`^\[FLOWCODE_Q\](\{.+\})$`)

// ScanLine 扫描每一行 stdout 输出，检测问询标记
func (d *QuestionDetector) ScanLine(line string) bool {
    matches := questionRe.FindStringSubmatch(line)
    if len(matches) < 2 {
        return false // 不是问询行
    }

    var input AIQuestionInput
    if err := json.Unmarshal([]byte(matches[1]), &input); err != nil {
        return false // JSON 解析失败，忽略
    }

    // 校验必填字段
    if input.Question == "" {
        return false
    }

    // 触发问询回调
    if d.onQuestion != nil {
        if err := d.onQuestion(&input); err != nil {
            // onQuestion 出错记录日志，不中断 AI 执行
            slog.Warn("question handler error", "err", err)
        }
    }

    return true
}

type AIQuestionInput struct {
    Question string   `json:"question"`
    Context  string   `json:"context"`
    Options  []string `json:"options"`
}
```

#### 答案写入与恢复执行

用户回答后，flowcode 将答案写入沙盒的 `/workspace/.flowcode_answer.md` 文件，然后恢复 AI 子进程：

```go
// 写入答案到沙盒
func (m *SandboxManager) WriteAnswer(ctx context.Context, issueID string, answerContent string) error {
    sb := m.getSandbox(issueID)
    if sb == nil {
        return fmt.Errorf("sandbox not found for issue %s", issueID)
    }

    writeCmd := fmt.Sprintf("cat > /workspace/.flowcode_answer.md << 'ANSWER_EOF'\n%s\nANSWER_EOF",
        answerContent)
    _, err := m.Exec(ctx, sb, []string{"sh", "-c", writeCmd}, nil)
    return err
}

// 继续 AI 子进程
func (a *OpenCodeAdapter) Continue(ctx context.Context, sb *Sandbox) error {
    // 向子进程发送 SIGCONT 信号
    cmd := exec.Command("docker", "kill", "--signal=SIGCONT", sb.ContainerID)
    return cmd.Run()
}
```

#### 配置项

| 配置键 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| enableAIQuestions | bool | true | 项目是否启用 AI 主动询问 |
| aiQuestionTimeout | int | 3600 | 问询超时自动跳过 (秒)，0 不设超时 |

## 7.2 Docker 沙盒执行环境

### 架构

```
┌─────────────────────────────────────────────────────────┐
│                   flowcode Server (Go)                  │
│  ┌───────────────────────────────────────────────────┐  │
│  │              SandboxManager                       │  │
│  │                                                  │  │
│  │  ┌────────────────┐  ┌────────────────┐          │  │
│  │  │ Container A    │  │ Container B    │  ...     │  │
│  │  │ (issue-abc123) │  │ (issue-def456) │          │  │
│  │  │                │  │                │          │  │
│  │  │ flowcode/      │  │ flowcode/      │          │  │
│  │  │ sandbox:latest │  │ sandbox:latest │          │  │
│  │  └────────────────┘  └────────────────┘          │  │
│  └──────────────┬────────────────────────────────────┘  │
│                 │ Docker SDK (containerd)               │
└─────────────────┼──────────────────────────────────────┘
                  │
         ┌────────┴────────┐
         │    dockerd      │
         │  (Unix Socket)  │
         └─────────────────┘
```

每个 Issue 执行时，`SandboxManager`：

1. 拉取/复用 `flowcode/sandbox:latest` 镜像（基础镜像至少包含 Git + Node/Python/Go；当前仓库默认预装 `opencode`，`codex` / `claude-code` 可按部署环境继续扩展）
2. 创建容器，挂载独立 volume `flowcode-sandbox-{issueId}` 作为 `/workspace`
3. 容器内 `git clone` 目标仓库到 `/workspace/repo`
4. 注入 Skills、分析摘要、执行计划到 `/workspace/.flowcode_prompt.md`
5. 容器内通过 CLI 调用 AI 工具（`opencode run --workspace /workspace/repo ...`）
6. 实时捕获 stdout/stderr → WebSocket 推送
7. 执行完毕：提交变更 → push → 销毁容器（`KeepOnError` 时保留现场）

### Docker 沙盒镜像

#### 基础镜像（Node 22 + Go + Python）

```dockerfile
# Dockerfile.sandbox
FROM node:22-bookworm-slim

ENV DEBIAN_FRONTEND=noninteractive \
    PIP_NO_CACHE_DIR=1 \
    PIP_DISABLE_PIP_VERSION_CHECK=1 \
    NPM_CONFIG_UPDATE_NOTIFIER=false \
    NPM_CONFIG_FUND=false \
    PNPM_HOME=/home/sandbox/.local/share/pnpm \
    PATH=/home/sandbox/.local/share/pnpm:/home/sandbox/.local/bin:${PATH}

RUN apt-get update && apt-get install -y --no-install-recommends \
    bash \
    ca-certificates \
    curl \
    git \
    golang-go \
    openssh-client \
    python3 \
    python3-pip \
    tini \
    && apt-get clean \
    && rm -rf /var/lib/apt/lists/*

RUN corepack enable
RUN curl -fsSL https://opencode.ai/install.sh | bash

# Optional AI CLI extensions:
# - codex: add the official Codex CLI install command here when needed.
# - claude-code: add the official Claude Code CLI install command here when needed.

RUN useradd --create-home --shell /bin/bash --uid 10001 sandbox \
    && mkdir -p /workspace \
    && chown -R sandbox:sandbox /workspace /home/sandbox

USER sandbox
WORKDIR /workspace

ENTRYPOINT ["/usr/bin/tini", "--"]
CMD ["sleep", "infinity"]
```

说明：
- 当前仓库的 `Dockerfile.sandbox` 采用 `node:22-bookworm-slim`，优先保证 Node.js 版本稳定，同时控制镜像体积。
- Node.js 生态默认可用 `node`、`npm`、`npx`，并通过 `corepack` 支持 `yarn` / `pnpm`。
- 当前仓库默认在基础镜像中预装 `opencode`。
- `codex`、`claude-code` 暂未直接安装，先以 Dockerfile 注释占位，后续按官方安装方式补充。

```bash
# 构建
docker build -f Dockerfile.sandbox -t flowcode/sandbox:latest .
```

#### 语言专用镜像（通过 `config/sandbox.yaml` 项目覆盖）

不同语言项目通过 `overrides.image` 指定预装对应运行时的镜像：

```yaml

overrides:
  "java-project":
    image: "flowcode/sandbox:java-21"     # 预装 JDK 21 + Maven + Gradle
  "rust-project":
    image: "flowcode/sandbox:rust-1.85"   # 预装 Rust + Cargo
  "dotnet-project":
```

```dockerfile
# Dockerfile.sandbox.java — Java 专用镜像
FROM ubuntu:22.04
RUN apt-get update && apt-get install -y --no-install-recommends \
    git curl wget ca-certificates openjdk-21-jdk maven \
    nodejs npm python3 python3-pip golang-go \
```

### SandboxManager 实现

```go
// internal/service/sandbox/docker.go

import (
    "github.com/docker/docker/client"
    "github.com/docker/docker/api/types"
    "github.com/docker/docker/api/types/container"
    "github.com/docker/docker/api/types/mount"
)

type DockerSandbox struct {
    cli     *client.Client
    image   string               // "flowcode/sandbox:latest"
    network string               // 容器网络模式
}

func NewDockerSandbox(cli *client.Client) *DockerSandbox {
    return &DockerSandbox{
        cli:     cli,
        image:   os.Getenv("SANDBOX_IMAGE"),  // 默认 flowcode/sandbox:latest
        network: "bridge",
    }
}

// Create 创建沙盒容器（不启动）
func (d *DockerSandbox) Create(ctx context.Context, id string, cfg SandboxConfig) (*Sandbox, error) {
    volumeName := fmt.Sprintf("flowcode-sandbox-%s", id)

    resp, err := d.cli.ContainerCreate(ctx,
        &container.Config{
            Image:      d.image,
            Cmd:        []string{"sleep", "infinity"},  // 保持运行
            WorkingDir: "/workspace",
            Env:        cfg.Env,
        },
        &container.HostConfig{
            Mounts: []mount.Mount{
                {
                    Type:   mount.TypeVolume,
                    Source: volumeName,
                    Target: "/workspace",
                },
            },
            Resources: container.Resources{
                Memory:   cfg.MemoryMB * 1024 * 1024,   // 2GB
                NanoCPUs: int64(cfg.CPUs * 1e9),        // 2 CPUs
                DiskQuota: cfg.MaxDiskMB * 1024 * 1024,  // 2GB disk
            },
            NetworkMode: container.NetworkMode(d.network),
            AutoRemove:  !cfg.KeepOnError,  // 正常自动删除
        },
        nil, nil, id,
    )
    if err != nil {
        return nil, fmt.Errorf("create container: %w", err)
    }

    return &Sandbox{
        ID:          id,
        ContainerID: resp.ID,
        VolumeName:  volumeName,
        WorkDir:     "/workspace",
    }, nil
}

// Exec 在沙盒容器内执行命令，流式收集日志
func (d *DockerSandbox) Exec(ctx context.Context, sb *Sandbox, cmd []string, onLog func(string)) (*ExecResult, error) {
    execConfig := types.ExecConfig{
        Cmd:          cmd,
        AttachStdout: true,
        AttachStderr: true,
        WorkingDir:   sb.WorkDir,
    }

    execResp, err := d.cli.ContainerExecCreate(ctx, sb.ContainerID, execConfig)
    if err != nil {
        return nil, fmt.Errorf("exec create: %w", err)
    }

    attachResp, err := d.cli.ContainerExecAttach(ctx, execResp.ID, types.ExecStartCheck{})
    if err != nil {
        return nil, fmt.Errorf("exec attach: %w", err)
    }
    defer attachResp.Close()

    // 流式读取
    scanner := bufio.NewScanner(attachResp.Reader)
    var fullOutput strings.Builder
    for scanner.Scan() {
        line := scanner.Text()
        fullOutput.WriteString(line + "\n")
        if onLog != nil {
            onLog(line)
        }
    }

    // 获取退出码
    inspectResp, _ := d.cli.ContainerExecInspect(ctx, execResp.ID)

    return &ExecResult{
        Output:   fullOutput.String(),
        ExitCode: inspectResp.ExitCode,
    }, nil
}

    d.cli.ContainerStop(ctx, sb.ContainerID, container.StopOptions{Timeout: &timeout})
    // 删除卷
    d.cli.VolumeRemove(ctx, sb.VolumeName, true)
    return nil
```

```go
// 三种调度策略
func newScheduler(strategy string) Scheduler {
    switch strategy {
    case "round_robin":
        return &RoundRobinScheduler{next: 0}
    case "least_containers":
        return &LeastContainersScheduler{}   // 默认：选容器数最少的节点
    default:
        return &RandomScheduler{}            // 随机
    }
}

// 最少容器数策略
func (s *LeastContainersScheduler) Select(clients []*client.Client) *client.Client {
    var best *client.Client
    minCount := int(^uint(0) >> 1)
    for _, cli := range clients {
        info, _ := cli.Info(context.Background())
        if info.ContainersRunning < minCount {
            minCount = info.ContainersRunning
            best = cli
        }
    }
    return best
}
```

```go
    }

    var result AnalyzeResult
    if err := json.Unmarshal(extractJSON(string(output)), &result); err != nil {
        return nil, fmt.Errorf("parse analyze result: %w", err)
    }

    // 校验
    categories := map[string]bool{
        "feature": true, "bug": true, "refactor": true,
        "docs": true, "infra": true, "test": true,
    }
    if !categories[result.Category] {
        return nil, fmt.Errorf("invalid category: %s", result.Category)
    }

    return &result, nil
}
```

### 异步分析任务

```go
// internal/queue/analyze_handler.go

func HandleAnalyzeTask(ctx context.Context, t *Task) error {
    var payload AnalyzePayload
    json.Unmarshal(t.Payload, &payload)

    // 1. 加载 Issue
    var issue model.Issue
    db.First(&issue, "id = ?", payload.IssueID)

    // 2. 加载项目（获取上下文）
    var project model.Project
    db.First(&project, "id = ?", issue.ProjectID)

    // 3. 调用 IssueAnalyzer（单次调用完成分析+分类）
    analyzer := &IssueAnalyzer{opencodePath: "opencode"}
    result, err := analyzer.Analyze(ctx, AnalyzeInput{
        Title:       issue.Title,
        Description: issue.Description,
        ProjectName: project.Name,
    })
    if err != nil {
        slog.Warn("AI analyze failed, keeping defaults", "issue", issue.ID, "err", err)
        return nil
    }

    // 4. 回写分析结果（isBug + needsCodeChange）
    db.Model(&issue).Updates(map[string]any{
        "is_bug":            result.IsBug,
        "needs_code_change": result.NeedsCodeChange,
        "analysis_reason":   result.AnalysisReason,
        "analysis_completed": true,
        "category":           result.Category,
        "category_source":    "ai",
        "category_confidence":  result.Confidence,
        "category_classified_at": time.Now(),
    })

    // 5. 发射事件
    hookRegistry.Emit(ctx, "issue:analyzed", map[string]any{
        "issue_id":         issue.ID,
        "is_bug":           result.IsBug,
        "needs_code_change": result.NeedsCodeChange,
        "category":         result.Category,
        "confidence":       result.Confidence,
    })

    // 6. 若 needsCodeChange=false，标记 Issue 跳过代码执行
    if !result.NeedsCodeChange {
        slog.Info("Issue does not need code change, skipping execution",
            "issue", issue.ID, "analysis", result.AnalysisReason)
    }

    return nil
}
```

### Go 实现（基于用户描述生成测试）

```go
// internal/service/test_runner.go

type TestRunner struct {
    db       *gorm.DB
    aiTool   adapter.AIToolAdapter
    sandbox  *SandboxManager
}

// Run 在 Docker 沙盒中执行测试代码，返回结构化结果
// 优先级: tc.TestScript (用户自定义) > framework 默认映射
func (r *TestRunner) Run(ctx context.Context, tc *model.TestCase) (*TestResult, error) {
    issue, _ := r.getIssue(ctx, tc.IssueID)
    project, _ := r.getProject(ctx, issue.ProjectID)

    // 1. 创建 Docker 沙盒（项目级可覆盖镜像，见 config/sandbox.yaml overrides）
    sb, err := r.sandbox.Create(ctx, project.OrgID+"/test-"+tc.ID)
    if err != nil {
        return nil, fmt.Errorf("create sandbox: %w", err)
    }
    defer r.sandbox.Cleanup(ctx, sb)

    // 2. 拉取项目代码
    if err := r.sandbox.PrepareWorkspace(ctx, sb, project); err != nil {
        return nil, fmt.Errorf("prepare workspace: %w", err)
    }

    // 3. 注入测试代码到容器内正确位置（先注入，{testFile} 变量依赖此路径）
    testPath := fmt.Sprintf("/workspace/repo/%s", tc.TestFilePath)
    writeCmd := []string{"sh", "-c", fmt.Sprintf(
        "cat > %s << 'EOTEST'\n%s\nEOTEST", testPath, tc.TestCode,
    )}
    if _, err := r.sandbox.Exec(ctx, sb, writeCmd, nil); err != nil {
        return nil, fmt.Errorf("inject test code: %w", err)
    }

    // 4. 解析脚本：custom testScript > framework 默认映射
    script := r.resolveScript(tc, testPath)

    // 5. 安装依赖
    if script.Install != "" {
        r.sandbox.Exec(ctx, sb, []string{"sh", "-c", script.Install}, nil)
    } else if depsCmd := r.getDepsInstallCmd(tc.Framework); depsCmd != nil {
        r.sandbox.Exec(ctx, sb, depsCmd, nil)
    }

    // 6. 执行测试命令
    cmd := []string{"sh", "-c", script.Command}
    result, err := r.sandbox.Exec(ctx, sb, cmd, nil)
    if err != nil {
        return nil, fmt.Errorf("run test: %w", err)
    }

    // 7. 解析结构化结果
    return r.parseResult(result.Output, tc.Framework), nil
}

// resolveScript 返回最终执行的脚本，custom > framework 默认
func (r *TestRunner) resolveScript(tc *model.TestCase, testPath string) Script {
    s := Script{
        Command: fmt.Sprintf("cd /workspace/repo && %s 2>&1", tc.Framework),
    }

    // 1. 用户自定义优先
    if tc.TestScript != nil && tc.TestScript.Command != "" {
        s.Command = tc.TestScript.Command
        if tc.TestScript.Install != "" {
            s.Install = tc.TestScript.Install
        }
        if tc.TestScript.Timeout > 0 {
            s.Timeout = tc.TestScript.Timeout
        }
    } else {
        // 2. 回退到 framework 默认映射
        cmds := r.buildTestCmd(tc.Framework)
        if len(cmds) > 2 {
            s.Command = cmds[2] // 取 sh -c 后面的命令
        }
        if deps := r.getDepsInstallCmd(tc.Framework); deps != nil && len(deps) > 2 {
            s.Install = deps[2]
        }
    }

    // 3. 变量替换
    s.Command = strings.NewReplacer(
        "{testFile}", testPath,
        "{workspace}", "/workspace/repo",
        "{projectDir}", filepath.Base(testPath[0:strings.LastIndex(testPath, "/")]),
    ).Replace(s.Command)
    s.Install = strings.NewReplacer(
        "{testFile}", testPath,
        "{workspace}", "/workspace/repo",
        "{projectDir}", filepath.Base(testPath[0:strings.LastIndex(testPath, "/")]),
    ).Replace(s.Install)

    // 4. 确保 stderr 重定向到 stdout
    if !strings.Contains(s.Command, "2>&1") {
        s.Command += " 2>&1"
    }

    return s
}

type Script struct {
    Install string
    Command string
    Timeout int
}

func (r *TestRunner) parseResult(output string, framework string) *TestResult {
    tr := &TestResult{
        Output: output,
        RunAt:  time.Now(),
    }

    // 通用解析：匹配 PASS/FAIL 行和覆盖率
    passRe := regexp.MustCompile(`--- PASS:`)
    failRe := regexp.MustCompile(`--- FAIL:`)
    covRe := regexp.MustCompile(`coverage: ([\d.]+)%`)

    tr.Passed = len(passRe.FindAllString(output, -1))
    tr.Failed = len(failRe.FindAllString(output, -1))
    tr.Total = tr.Passed + tr.Failed

    if matches := covRe.FindStringSubmatch(output); len(matches) > 1 {
        tr.Coverage = matches[1] + "%"
    }

    if tr.Failed == 0 && tr.Total > 0 {
        tr.Status = "passed"
    } else {
        tr.Status = "failed"
        // 提取失败详情
        tr.Errors = r.extractFailureDetails(output)
    }

    return tr
}

func (r *TestRunner) extractFailureDetails(output string) []TestError {
    re := regexp.MustCompile(`(\S+):(\d+):\s*(.+)`)
    var errors []TestError
    for _, match := range re.FindAllStringSubmatch(output, -1) {
        line, _ := strconv.Atoi(match[2])
        errors = append(errors, TestError{
            File:    match[1],
            Line:    line,
            Message: match[3],
        })
    }
    return errors
}

// TestResult + TestError 定义见 03-数据模型.md 中 `TestCase.result` 与 `TestResult` 结构说明
```
