  ├── API Key
  ├── 钩子配置
  └── 通用设置
─────────────

  ├── 设备管理
  └── 登录日志
```

- 侧边栏顶部显示当前项目名称与切换入口（下拉选择项目）
- 项目切换器下方显示当前组织名称，多组织用户可切换组织
- 底部显示当前用户信息与退出按钮
- 组织管理仅在组织管理员角色时显示
- 移动端折叠为汉堡菜单

### 12.2.2 全局状态

| 状态类型 | 说明 |
|---------|------|
| 加载中 | 全局顶栏细进度条 (NProgress 风格) |
| 网络错误 | Toast 通知 + 重试按钮 |
| 未授权 (401) | 自动跳转登录页 |
| 无权限 (403) | 展示 403 禁止访问页面 |
| 服务端错误 (500) | Toast 通知 + 错误详情折叠面板 |
| WebSocket 断开 | 状态指示灯变红 + 自动重连（指数退避，最多 5 次） |

### 12.2.3 国际化

- 默认语言跟随浏览器 `navigator.language`，用户可在个人设置中覆盖
- 所有页面文案通过 `vue-i18n` 管理，不硬编码
- 日期时间格式跟随语言环境

## 12.3 页面详细需求

### 12.3.1 登录页 `/login`

**数据来源**: `POST /api/v1/auth/login`

**页面结构**:
```
┌──────────────────────────────┐
│        flowcode Logo         │
│        AI 驱动的需求管理       │
│                              │
│   ┌──────────────────────┐   │
│   │ 邮箱                  │   │
│   └──────────────────────┘   │
│   ┌──────────────────────┐   │
│   │ 密码                  │   │
│   └──────────────────────┘   │
│   ┌──────────────────────┐   │
│   │ 登录                  │   │
│   └──────────────────────┘   │
│                              │
│   没有账号？注册             │
└──────────────────────────────┘
```

**交互**:
- 输入邮箱 + 密码 → 点击登录 → 成功跳转 `/dashboard`
- 支持 Enter 键提交

**状态**:
| 状态 | 表现 |
|------|------|
| 加载中 | 登录按钮显示 spinner + 禁用 |
| 验证失败 | 表单字段标红 + 错误提示（"邮箱或密码错误"） |
| 账号锁定 | "账号已被锁定，请 15 分钟后重试" |
| 网络错误 | "网络连接失败，请检查网络" |
| 空值提交 | 邮箱/密码必填校验 |

**边界场景**:
- 已登录用户访问 `/login` → 自动跳转 `/dashboard`
- 登录成功后存储 JWT Token 到 localStorage，后续请求自动附带 `Authorization: Bearer <token>`
- Token 过期 (401) → 清除本地 token → 跳转登录页

### 12.3.4 创建/编辑项目 `/projects/create`

**数据来源**: `POST /api/v1/projects` / `PUT /api/v1/projects/:id`

**页面结构**:
```
┌──────────────────────────────────────────────┐
│  创建项目                                    │
├──────────────────────────────────────────────┤
│  项目名称 *           [________________]      │
│  项目描述             [________________]      │
│  项目标识 (slug)     [________________]      │
│                                              │
│  ── Git 配置 ──                              │
│  Git 仓库地址         [________________]      │
│                      [校验地址]  → 校验状态    │
│  选择凭证             [下拉选择 ▼]            │
│  默认分支             [main           ▼]     │
│                                              │
│  ── 高级配置 (折叠) ──                        │
│  AI 自动执行          [开关]                  │
│  默认 AI 工具         [claude-code    ▼]     │
│  PR 自动创建          [开关] 默认开启          │
│  分支命名模板         [flowcode/{id}-{slug}]  │
│                                              │
│  [取消]  [创建项目]                           │
└──────────────────────────────────────────────┘
```

**交互**:
- 输入项目名称 → 自动生成 slug（支持手动修改）
- 输入 Git URL → 点击"校验地址" → 调用 `POST /api/v1/git/validate-url`
- Git 凭证下拉列表来自 `GET /api/v1/projects/:pid/git/credentials`
- 若无可用凭证，提示"请先配置 Git 凭证"，并提供快捷入口

**校验状态 (Git URL)**:
| 状态 | 表现 |
|------|------|
| 校验中 | 按钮 spinner + "正在校验..." |
| 校验成功 | 绿色勾 + "校验成功" + 显示默认分支和分支列表 |
| 校验失败 | 红色叉 + 错误信息（"无法访问仓库: ..."） |
| 未校验 | 灰色提示"建议校验仓库地址" |

**边界场景**:
- 创建时可暂不配置 Git（允许纯需求管理模式），后续编辑补配
- 编辑模式下若修改 Git URL，需重新校验
- 凭证已过期时有红色标签提示"凭证已过期"
- 表单未保存离开时弹出"有未保存的更改"确认

│                                                   │
│  ☑ Issue 1: 实现 JWT 认证中间件        P0 feature │
│    └ Claude Code · 核心阻塞，先完成                │

│    └ OpenCode · 最后执行                          │
│                                                   │
│  [全选] [确认拆解] [手动调整]    [重新拆解]        │
├──────────────────────────────────────────────────┤
│  ── 关联 Issues ──                                │
│  ┌──────────────────────────────────────────┐    │
│  │ iss_001 实现 JWT 认证中间件   running     │    │
│  │ iss_002 编写登录/注册 API     queued      │    │
│  └──────────────────────────────────────────┘    │
├──────────────────────────────────────────────────┤
│  ── 操作按钮 ──                                   │
│  [审批通过] [驳回] [AI 拆解] [编辑] [删除]         │
└──────────────────────────────────────────────────┘
```

**交互**:
- AI 拆解：点击"AI 拆解" → 触发 `POST /api/v1/requirements/:id/ai-decompose` → 异步返回拆解结果
- 用户可勾选/取消子 Issue，修改优先级和 AI 工具选择
- 确认拆解 → `POST /api/v1/requirements/:id/convert-to-issues` → 批量创建 Issue
- 审批通过 → `POST /api/v1/requirements/:id/approve`
- 驳回 → 弹窗输入驳回理由 → `POST /api/v1/requirements/:id/reject`

**状态**:
| 状态 | 表现 |
|------|------|
| AI 拆解中 | 拆解结果区域显示骨架屏 + "AI 正在分析需求..." |
| 拆解失败 | "AI 拆解失败" + 错误原因 + 重试按钮 |
| 拆解返回空 | "AI 无法拆解此需求，请手动创建 Issue" |
| 已拆解但未确认 | 黄色提示条"拆解结果尚未确认" |

**边界场景**:
- 需求已关联 Issue 后不可删除
- 驳回后的需求可重新编辑提交
- 拆解结果超过 10 个时截断并提示

### 12.3.23 审计日志 `/projects/:id/audit-log`

**数据来源**: `GET /api/v1/projects/:id/audit-logs`

**概述**: 记录项目内所有关键操作——Issue 状态变更、审批、执行、Git 操作、凭证管理、权限变更等。每条记录包含操作人、操作类型、目标对象、变更详情、时间戳和 IP。

**操作类型**（参考 04-API设计 和 08-Git集成方案 8.13）:
| 操作分类 | action 示例 |
|---------|-----------|
| Issue | issue.create / issue.approve / issue.reject / issue.execute / issue.pause / issue.retry / issue.close / issue.reopen |
| 需求 | requirement.create / requirement.approve / requirement.reject / requirement.decompose |
| Git | git.connect / git.disconnect / git.import-issues / git.create-pr |
| 项目 | project.create / project.update / project.delete / project.archive |
| 权限 | member.invite / member.remove / member.role_change |
| 凭证 | apikey.create / apikey.revoke |

**页面结构**:
```
┌──────────────────────────────────────────────────────────────┐
│  审计日志                                      [导出 CSV]     │
│  [时间范围: 最近 7 天 ▼] [操作类型 ▼] [操作人 ▼] [搜索...]    │
├──────────────────────────────────────────────────────────────┤
│  时间         操作人   IP              操作           目标     │
│  ──────────   ──────  ──────          ────           ────    │
│  05-10 10:30  张三    192.168.1.100   issue.approve  ISS-001 │
│    └ 详情: draft → approved，审核通过                  [展开] │
│  05-10 10:28  李四    10.0.0.5        issue.create   ISS-002 │
│    └ 详情: 创建 Issue "修复登录页面样式错乱"           [展开] │
│  05-10 09:00  张三    192.168.1.100   git.connect    —       │
│    └ 详情: 添加 GitHub 凭证 "GitHub 主仓库"           [展开] │
│  05-10 08:00  系统    —               issue.execute  ISS-001 │
│    └ 详情: 开始执行，AI 工具: Claude Code             [展开] │
├──────────────────────────────────────────────────────────────┤
│  共 1,247 条  [< 1 2 3 ... 25 >]   每页 50 条               │
└──────────────────────────────────────────────────────────────┘
```

**交互**:
- 点击行 → 展开操作详情 JSON（变更前后字段对比）
- 时间范围快捷选择：最近 1 小时 / 今天 / 最近 7 天 / 最近 30 天 / 自定义
- 操作人筛选支持多选
- 搜索支持按目标 ID（如 ISS-001）或关键词
- 「导出 CSV」按钮导出当前筛选结果

**状态**:
| 状态 | 表现 |
|------|------|
| 加载中 | 表格骨架屏 |
| 空结果 | "暂无审计日志" |
| 筛选无结果 | "当前筛选条件下无匹配记录" + 清除筛选按钮 |
| 加载失败 | "审计日志加载失败" + 重试按钮 |

**边界场景**:
- 审计日志保留 2 年，超过保留期的数据显示"已归档，请联系管理员"
- 系统自动操作（如 AI 分析回写、自动执行）的操作人显示为「系统」
- 导出 CSV 超过 10000 条时分批下载
- 项目归档后审计日志仍可查看（只读）

## 12.5 跨页面交互流程

### 12.5.1 需求 → Issue → 执行 → PR 闭环
```

```
监控面板 → 事件列表 → 转为 Issue → Issue 看板

### 12.5.3 Skills → Issue 执行闭环
```
Skills 管理 → 安装/配置 Skill → Issue 绑定 Skill
    → AI 执行时注入 Skill 指令 → 影响代码生成质量
```

## 12.6 技术实现要点

### 12.6.1 路由设计
- Vue Router 4 嵌套路由
- 项目级路由守卫：`/projects/:id` 下所有子路由需先加载项目上下文
- 权限守卫：组织管理员才能访问组织管理页面

### 12.6.2 状态管理 (Pinia)
- projectStore: 当前项目信息、设置、凭证
- issueStore: Issue 列表、看板状态、当前 Issue、执行日志
- userStore: 用户信息、JWT、权限
- monitorStore: 监控事件、统计

### 12.6.3 WebSocket 管理
- 执行日志 WebSocket：`/ws/execution/:issueId`
- 自动重连策略：1s → 2s → 4s → 8s → 16s（最多 5 次）
- 组件卸载时自动断开

### 12.6.4 错误处理统一策略
- API 调用统一拦截，401 自动跳转登录
- 统一响应格式 `{code, data, message, timestamp}`
- Toast 通知（成功/警告/错误三种类型）
- 关键操作失败需模态框提示

### 12.6.5 表单校验
- Element Plus 表单校验规则
- Git URL 格式校验（仅允许 HTTP/HTTPS/SSH）
- API Key 权限校验
- Skill 内容安全校验（前端预检危险指令）
