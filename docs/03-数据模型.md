**category 自动分类方案**（混合：关键词计分 + AI 异步补充分类）：

```
Issue 创建 → 关键词计分（同步，0ms）
                │
        ┌───────┴────────┐
        ▼                ▼
   高置信度(≥0.8)     低置信度 / 未命中
        │                │
        ▼                ▼
   直接采用         投递 Asynq → opencode classify
                        │
                        ▼
                   AI 分类结果异步回写
```

**关键词计分规则**（由 Classifier 模块执行）：

| 关键词 | 分类 | 权重 |
|--------|------|:---:|
| bug / 修复 / bugfix / 报错 / 异常 / 崩溃 / 错误 / error / crash | bug | 2.0 |
| 新增 / 添加 / 增加 / 实现 / 开发 / 功能 / feature / add | feature | 2.0 |
| 重构 / 优化 / 改进 / 整理 / 清理 / refactor / improve | refactor | 2.0 |
| 文档 / 说明 / 注释 / readme / docs / document | docs | 2.0 |
| 部署 / ci / cd / 环境 / 配置 / 基础设施 / deploy / infra / chore | infra | 2.0 |
| 测试 / 用例 / test / spec | test | 2.0 |
| 修复 / 改进 (弱匹配) | bug / refactor | 1.0 |
| 升级 / 迁移 / upgrade / migrate | infra | 1.0 |

> 英文关键词通过 strings.ToLower 统一处理。多关键词权重累加，最高分对应分类。
> 置信度 = 最高分 / (最高分 + 次高分)，≥0.8 直接采用，<0.8 触发 AI 异步补充分类。

**status 字段说明**：

| 状态 | 含义 |
|------|------|

| approved | 审核通过，可触发 AI 生成测试代码 |

| generated | 测试代码已生成，待运行 |
| passed | 测试运行通过 |
| failed | 测试运行失败 |

**result JSONB 字段结构**：

```go
// 存储到 TestCase.result (JSONB)
type TestResult struct {
    Status     string      `json:"status"`     // passed / failed
    DurationMs int64       `json:"durationMs"` // 执行耗时 (毫秒)
    Total      int         `json:"total"`      // 测试总数
    Passed     int         `json:"passed"`     // 通过数
    Failed     int         `json:"failed"`     // 失败数
    Skipped    int         `json:"skipped"`    // 跳过数
    Coverage   string      `json:"coverage"`   // 覆盖率 "82.5%"
    Output     string      `json:"output"`     // 测试原始输出 (截取前 10KB)
    Errors     []TestError `json:"errors"`     // 失败详情
    RunAt      time.Time   `json:"runAt"`      // 执行时间
}

type TestError struct {
    File    string `json:"file"`    // 失败文件
    Line    int    `json:"line"`    // 失败行号
    Message string `json:"message"` // 失败信息
}
```

### 3.2.5c HistoryRecord 历史记录表

所有核心实体 (Project / Requirement / Issue / TestCase) 的变更均自动记录到该表，支持时间线查看和版本对比。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | VARCHAR(36) PK | UUID 主键 |
| targetType | VARCHAR(20) | 实体类型: project / requirement / issue / test_case |
| targetId | VARCHAR(36) | 实体 ID |
| action | VARCHAR(50) | 操作类型: created / updated / status_changed / deleted / reviewed |
| fieldName | VARCHAR(50) | 变更字段 (整体变更时为 "*") |
| oldValue | JSON | 变更前值 (NULL=新建) |
| newValue | JSON | 变更后值 |
| changedBy | VARCHAR(36) FK→User | 操作人 |
| createdAt | TIMESTAMP | 记录时间 |

**action 枚举说明**：

| action | 触发场景 |
|--------|---------|
| created | 实体创建 |
| updated | 字段修改 (title/description/priority 等) |
| status_changed | 状态流转 (draft→approved→running→done) |
| deleted | 实体软删除 |
| reviewed | 审核操作 (approve/reject) |
| test_generated | AI 生成测试代码完成 |
| test_executed | 测试运行完成 |

  "testCoverageTarget": 80         // 默认覆盖率目标(%)

### Requirement.status 状态枚举

| 状态 | 含义 |
|------|------|
| draft | 草稿 |
| reviewing | 审核中 |
| approved | 已审核通过，可拆解 |
| in_progress | 拆解的 Issue 执行中 |
| done | 全部 Issue 完成 |
| archived | 归档 |

## 3.4 索引策略

```sql
-- Issue 高频查询索引
CREATE INDEX idx_issue_project_status ON issues(project_id, status);
CREATE INDEX idx_issue_requirement ON issues(requirement_id);
CREATE INDEX idx_issue_priority ON issues(project_id, priority);
CREATE INDEX idx_issue_category ON issues(project_id, category);

-- ExecutionLog 索引
CREATE INDEX idx_log_issue ON execution_logs(issue_id, attempt);

-- MonitorEvent 去重索引
CREATE INDEX idx_monitor_dedup ON monitor_events(project_id, dedup_hash, created_at);
CREATE INDEX idx_monitor_type ON monitor_events(project_id, event_type, created_at);

-- Requirement 索引
CREATE INDEX idx_req_project_status ON requirements(project_id, status);

-- TestCase 索引
CREATE INDEX idx_test_issue ON test_cases(issue_id, status);
CREATE INDEX idx_test_review ON test_cases(review_status, created_at);

-- HistoryRecord 索引 (时间线查询 + 实体对比)
CREATE INDEX idx_history_target ON history_records(target_type, target_id, created_at DESC);
CREATE INDEX idx_history_changed_by ON history_records(changed_by, created_at DESC);

-- Project 索引
CREATE INDEX idx_project_org ON projects(org_id, slug);

-- GitCredential 索引
CREATE INDEX idx_gitcred_project ON git_credentials(project_id, auth_method);
```
