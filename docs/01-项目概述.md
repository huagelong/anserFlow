# 01 - 项目概述

## 1.1 flowcode 是什么

flowcode 是一套专为 **vibe coding** 工作流设计的 SaaS 需求管理系统。它填补了「灵感 → 需求 → AI编码 → PR → 上线 → 监控反馈」全链路的工具空白，让独立开发者和微型创意团队可以像聊天一样管理整个软件开发周期。

核心理念：**用对话驱动开发，用自动化消除繁琐**。

flowcode 同时提供 **CLI 工具**、**RESTful API**、**Skills 技能包** 三种使用形态，满足从终端极客到 Web 用户的不同偏好。

## 1.2 目标用户

| 用户角色 | 典型场景 |
|---------|---------|
| 独立开发者 | 一个人维护多个 side project，希望快速把想法变成代码并部署 |
| 小型创意团队 (2-5人) | 快速迭代 MVP，非技术人员也能参与需求输入 |
| Indie Hacker | 验证 idea → 快速上线 → 收集反馈 → 持续迭代 |
| CLI 重度用户 | 终端内完成需求提交、触发执行、查看日志，不离开命令行 |
| API 集成者 | 通过 API Key 将 flowcode 嵌入现有 CI/CD 或自动化流程 |

## 1.3 解决的核心痛点

1. **需求碎片化**：灵感散落在聊天记录、备忘录、脑图中，缺乏统一入口
2. **编码执行断层**：提出 Issue 后仍需人工编码，AI 工具缺少调度中枢
3. **反馈闭环断裂**：线上 bug / 用户反馈无法自动回流到需求池
4. **工具链割裂**：需求管理、AI 编码、Git 管理、监控各自独立
5. **测试滞后**：AI 生成代码后缺乏自动测试验证，质量难以保证
6. **多租户缺失**：个人项目和企业项目杂糅，缺乏组织和权限隔离

## 1.5 核心竞争力

### 顺滑的需求到代码体验
用户只需用自然语言描述需求，系统自动拆解为可执行的 Issue，管理员一键审核后由 AI 自动编码。

### 开放式插件 + Skills 体系
AI 编程工具 (opencode / codex / claude code) 和 Git 平台 (GitHub / Gitea) 均通过标准 Hook 接口接入。Skills 以 Markdown 文件定义可复用的编码技能，AI 执行时自动加载。CLI 和 API 提供完整编程化访问能力。

### 全链路闭环
从需求提出到代码合并、从部署上线到监控反馈，所有环节都在一个系统内完成，数据不落地。

## 1.8 三种使用形态

### Web 控制台
基于 Vue 3 + Element Plus 的现代化管理界面，看板、甘特图、实时日志流一应俱全。

### CLI 工具 (`flowcode`)
```bash
# 终端内完成全流程
flowcode login
flowcode issue create "修复登录页面样式错乱" --priority p0 --tool opencode
flowcode issue watch abc123          # 实时查看执行日志
flowcode skill install go-best-practices
```

### REST API
```bash
# API Key 鉴权，可嵌入 CI/CD
curl -H "X-API-Key: fc_sk_xxx" \
  https://flowcode.example.com/api/v1/issues \
  -d '{"title":"自动构建失败修复","project_id":"p1","priority":"p0"}'
```

## 1.9 项目命名

**flowcode** = flow (流动/工作流) + code (编码)

寓意：灵感如流水般顺畅地转化为代码，享受 vibe coding 的心流体验。
