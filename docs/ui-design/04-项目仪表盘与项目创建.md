# 项目仪表盘、项目列表与创建项目

覆盖页面：

- `/`：Dashboard.vue
- `/projects`：ProjectList.vue
- `/projects/new`：ProjectCreate.vue

## 仪表盘

### 页面目标

让用户在 10 秒内判断项目健康度：Issue 是否堆积、PR 是否待处理、测试覆盖率是否达标、监控是否有异常、AI 执行队列是否卡住。

### 桌面布局

```
标题区：项目名称 / Git 状态 / 默认分支 / 时间范围
指标区：4 个统计卡片
主区左：最近 Issue + 执行队列
主区右：PR 状态 + 测试覆盖率 + 监控摘要
```

内容宽度：

- 1200px 以下单列。
- 1200px 以上两列：左侧 2fr，右侧 1fr。

### 统计卡片

卡片高 112px，四列栅格，间距 16px。

指标：

- Open Issues。
- In Review / Waiting Input。
- Tests pass rate / coverage。
- Open PRs。

每个卡片：

- 标题 13px，弱文字。
- 数值 28px，字重 700。
- 趋势或说明 12px。
- 状态异常时右上角显示警告点。

### 最近 Issue

列表显示 8 条：

- 标题。
- 状态徽章。
- 优先级。
- 分类与来源 rule / ai / manual。
- 最近更新时间。
- 下一步动作：审批、回答、查看日志、创建 PR。

行高 56px，点击进入 Issue 详情。

### 执行队列

展示 queued / running / waiting_input：

- running 显示 AI 工具、开始时间、尝试次数。
- waiting_input 显示待回答数量和“回答”按钮。
- queued 显示排队位置。

### PR 状态

显示 open / merged / closed 数量和最近 5 个 PR。

### 测试摘要

展示：

- 通过率。
- 覆盖率。
- 最近失败测试。
- 跳转测试用例页面。

### 监控摘要

展示最近 7 天：

- 总事件。
- 自动创建 Issue 数。
- 未处理错误。
- “查看监控”入口。

## 项目列表

### 页面目标

组织内项目扫描与进入。支持新建项目，不做复杂项目组合管理。

### 布局

标题栏：

- 标题：项目。
- 右侧主按钮：新建项目。

工具栏：

- 搜索项目名 / slug / Git 仓库。
- Provider 筛选：GitHub / GitLab / Gitea。
- Git 状态筛选：正常 / 未配置 / 验证失败。

项目列表默认用表格，移动端改卡片。

列：

| 列 | 宽度 | 说明 |
|----|------|------|
| 项目 | 280px | 名称、slug、描述摘要。 |
| Git | 260px | provider、repoId、默认分支。 |
| 健康 | 220px | open Issue、测试覆盖率、PR 数。 |
| 最近活动 | 160px | 更新时间。 |
| 操作 | 160px | 进入、设置、删除。 |

空态：

- 无项目：显示“创建第一个项目”，主按钮进入 `/projects/new`。

## 创建项目

### 页面目标

把文档定义的创建流程拆成可验证步骤：基础信息 → Git 凭证 → Git URL 校验 → 默认分支 → 项目设置 → 完成。

### 步骤条

顶部水平步骤条，桌面展示完整文字，移动端只显示当前步骤。

1. 基础信息。
2. Git 凭证。
3. 仓库校验。
4. 默认设置。
5. 完成。

### Step 1：基础信息

字段：

| 字段 | 控件 | 规则 |
|------|------|------|
| 项目名称 | 输入框 | 必填，最多 100 字。 |
| slug | 输入框 | 根据名称自动生成，可编辑。 |
| 描述 | Markdown 文本域 | 最小 120px 高。 |
| Git Provider | 单选卡 | github / gitlab / gitea。 |
| Git 协议 | 分段控件 | http / ssh。 |

### Step 2：Git 凭证

如果已有凭证：

- 下拉选择凭证。
- 显示凭证 provider、authMethod、过期时间或 SSH 指纹。

新增凭证：

- 名称。
- HTTP PAT 或 SSH 私钥，按协议动态显示。
- SSH 私钥使用多行安全输入框，默认隐藏。

保存后返回凭证 ID，继续下一步。

### Step 3：仓库校验

字段：

- Git 仓库 URL。
- 凭证选择。

动作：

- 主按钮：校验仓库。
- 调用 `POST /api/v1/git/validate-url`。

校验成功：

- 显示绿色状态条。
- 展示 repoId、defaultBranch、branches。
- 启用下一步。

校验失败：

- 显示错误详情。
- 禁止创建项目。
- 保留用户输入方便修改。

### Step 4：默认设置

只展示 `Project.settings` 中定义项：

| 设置 | 控件 | 默认 |
|------|------|------|
| autoExecute | Switch | false |
| autoCreatePR | Switch | true |
| defaultAITool | Select | claude-code |
| requireApproval | Switch | true |
| testRequireApproval | Switch | true |
| enableAIQuestions | Switch | true |
| aiQuestionTimeout | NumberInput | 3600 秒 |
| maxConcurrentExecutions | NumberInput | 3 |
| branchNaming | 输入框 | `flowcode/{{issueId}}-{{slug}}` |
| testFramework | Select | vitest |
| testCoverageTarget | Slider + NumberInput | 80 |

### Step 5：完成

创建成功后展示：

- 项目名称。
- Git 仓库。
- 默认分支。
- Git 凭证名称。
- 下一步按钮：进入仪表盘、创建第一个 Issue、配置 Skills。

### 体验细节

- 离开未完成流程时提示确认。
- 每一步内容宽度 760px，表单不撑满全屏。
- 成功创建后不展示密钥明文。
- Git 校验 loading 时显示“正在连接仓库”，按钮禁用。
