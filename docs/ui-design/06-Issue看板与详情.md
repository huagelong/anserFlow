# Issue 看板与详情页面设计

覆盖页面：

- `/projects/:pid/issues`：Issues.vue
- `/projects/:pid/issues/:iid`：IssueDetail.vue

## Issue 看板

### 页面目标

按状态管理 AI 编码工作流，让用户快速发现待审核、执行中、等待回答、失败和待 PR 的 Issue。

### 顶部区域

标题栏：

- 标题：Issue 看板。
- 副标题：显示当前项目、打开 Issue 数和待回答问询数。
- 主按钮：新建 Issue。
- 次按钮：导入 Git Issues。
- 顶栏居中提供全局快捷入口：`+ Issue  Ctrl I`，点击或按 `Ctrl+I` 打开新建 Issue 抽屉。

工具栏：

- 搜索：标题、描述、标签。
- 状态筛选。
- 优先级筛选。
- 分类筛选：bug / feature / refactor / docs / infra / test。
- AI 工具筛选：opencode / codex / claude-code。
- 来源筛选：flowcode / github / gitlab / gitea。
- 状态 Tabs：全部 / 草稿 / 已批准 / 排队 / 执行中 / 暂停 / 待回答 / 完成 / PR / 已结束。

### 状态 Tabs

Issue 看板默认使用状态 Tabs，而不是多列横向看板。状态数量较多时，多列看板会压缩单列宽度；Tabs 让用户一次聚焦一个状态，下方用横向行展示更多字段。

Tab 规则：

- 默认选中 `全部`，展示所有状态的 Issue 行。
- 每个 Tab 显示状态名和数量。
- 切换 Tab 后保留搜索、优先级、分类、AI 工具和来源筛选。
- `全部` Tab 展示所有状态内容，默认按最近更新时间排序。
- `PR` Tab 聚合 `pr_ready`、`pr_created`。
- `已结束` Tab 聚合 `deployed`、`closed`、`rejected`、`cancelled`、`failed`。

### 行列表

Tab 下方展示列表行。行高 46px，表头 30px，列表容器使用白色表面、浅边框和 6px 圆角。

列结构：

| 列 | 宽度 | 内容 |
|----|------|------|
| Issue / 摘要 | 自适应 | 标题 + 一行状态摘要。 |
| 状态 / 优先级 / 分类 | 190px | 当前状态、P0-P3、分类。 |
| 来源 / 执行方 | 140px | flowcode、Git 平台、AI 工具或 worker。 |
| 执行信号 | 160px | AI 分析、排队位置、运行时长、问询数、PR 状态等。 |
| 下一步 | 120px | 当前最推荐动作。 |

行交互：

- 点击标题或摘要区域进入 Issue 详情。
- 点击“下一步”只触发行内动作，不进入详情。
- 行内只展示一个主动作，其余动作进入更多菜单。
- 状态变更后当前 Tab 局部刷新，并保留滚动位置和筛选条件。

### 分页

行列表底部固定展示分页条：

- 左侧显示总数和每页数量，例如 `共 17 条 · 每页 10 条`。
- 右侧显示上一页、当前页、总页数、下一页，例如 `1 / 2`。
- 切换状态 Tab、筛选或搜索后，分页回到第一页。
- 切页时保留当前 Tab 和筛选条件，只刷新行列表区域。

### 状态行样式

每个状态复用同一行结构，用浅底色和强调色区分当前阶段：

| Tab | 包含状态 | 行底色 | 执行信号示例 | 下一步 |
|-----|----------|--------|--------------|--------|
| 草稿 | `draft` | 白色 | AI 分析完成 | 审批 |
| 已批准 | `approved` | 白色 | 计划可生成 | 执行 / 入队 |
| 排队 | `queued` | warning-light | 队列 2 / 3 | 查看队列 |
| 执行中 | `running` | primary-light | 已运行 42m | 查看日志 |
| 暂停 | `paused` | info-light | 等待恢复 | 恢复 / 取消 |
| 待回答 | `waiting_input` | warning-light | 2 个问询 | 回答问询 |
| 完成 | `done` | success-light | 可创建 PR | 创建 PR |
| PR | `pr_ready`, `pr_created` | primary-light | 等待创建 PR / 等待合并 | 创建 PR / 标记部署 |
| 已结束 | `deployed`, `closed`, `rejected`, `cancelled`, `failed` | danger-light 或 info-light | 错误已归档 / 可重开 | 重试 / 编辑 / 重开 |

状态表达不能只依赖颜色，必须同时显示状态文字。

### 状态动作守卫

状态变更只允许状态机可用动作，不能通过拖拽自由移动。

交互：

- “下一步”按钮来自当前状态允许的主动作。
- 更多菜单只列出当前状态可用的次动作。
- 不可用动作不展示；如需解释，在详情页右侧工作流栏展示原因。
- 危险动作如 close / reject / cancel 必须二次确认。

### 新建 Issue

使用右侧抽屉，宽 640px。

字段：

| 字段 | 控件 | 规则 |
|------|------|------|
| 标题 | 输入框 | 必填，最多 500 字。 |
| 描述 | Markdown 编辑器 | 最小高度 260px。 |
| 优先级 | 单选 | p0 / p1 / p2 / p3。 |
| AI 工具 | 工具卡片单选 | opencode / codex / claude-code。 |
| Skill | 多选 | 从已发布或已安装 Skill 中选择。 |
| 预估工时 | NumberInput | 可选。 |
| 指派人 | Select | 可选。 |
| 标签 | 多选或创建 | 使用项目标签。 |

提交后状态为 `draft`，并触发自动分析。创建成功后抽屉关闭，若当前在草稿 Tab，则新行插入列表顶部并短暂高亮。

### 全量列表视图

状态 Tabs 适合按阶段聚焦处理；全量列表视图适合跨状态筛选和批量扫描。列：

- 标题。
- 状态。
- 优先级。
- 分类。
- AI 分析。
- AI 工具。
- 指派人。
- PR。
- 更新时间。
- 操作。

## Issue 详情

### 页面目标

承载一次 Issue 的完整生命周期：描述、AI 分析、审批、计划、执行日志、问询、评论、测试、历史、PR。

### 布局

桌面端两栏：

```
左侧主内容：标题 + Tabs
右侧工作流栏：状态、下一步动作、元信息、PR/Git 摘要
```

- 右侧栏宽 360px。
- 主内容最小宽 720px。
- 移动端右侧栏折叠到标题下方，Tabs 单列。

### 详情头部

包含：

- Issue 标题。
- 状态、优先级、分类、来源。
- ID、创建者、创建时间。
- 若来自 Git 平台，显示原始 Issue 外链。

标题可内联编辑：

- 点击编辑按钮进入标题输入。
- 保存调用 `PUT /api/v1/issues/:id`。
- running / queued 状态下仍允许补充描述，但 AI 工具字段只在 approved 前可改。

### 右侧工作流栏

顶部显示当前状态和下一步动作。

动作映射：

| 当前状态 | 主动作 | 次动作 |
|----------|--------|--------|
| draft | 审批通过 | 驳回、编辑、删除 |
| approved | 执行 | 生成/重新生成计划、编辑 |
| queued | 取消执行 | 查看队列 |
| running | 暂停 | 取消、查看日志 |
| paused | 恢复执行 | 取消 |
| waiting_input | 回答问询 | 跳过、取消 |
| done | 创建 PR | 重试、关闭 |
| failed | 重试 | 编辑、关闭 |
| cancelled | 重试 | 关闭 |
| pr_ready | 创建 PR | - |
| pr_created | 标记已部署 | 重开 |
| deployed | 关闭 | 重开 |
| closed | 重新打开 | - |
| rejected | 重新打开 | 编辑、删除、关闭 |

危险操作：

- cancel、close、delete、reject 需要确认。
- fast_exec 只在管理员且符合条件时显示在更多菜单中。

### Tabs

#### 概览

区块：

- 描述 Markdown。
- AI 分析：isBug、needsCodeChange、analysisReason、analysisCompleted。
- 分类：category、source、confidence、classifiedAt。
- 关联需求。
- 标签与指派人。

#### 执行计划

显示：

- planSummary。
- planDetail Markdown。
- planGeneratedAt。
- 操作：生成计划 / 重新生成计划。

未生成时：

- approved 状态显示主操作“生成执行计划”。
- 其他状态说明计划会在执行前自动补齐或当前状态不允许生成。

#### 执行日志

嵌入执行日志组件，详见 `07-AI问询执行日志与历史.md`。

#### AI 问询

显示当前 Issue 的问询列表摘要：

- pending 问询置顶。
- 每条显示问题、上下文、选项、回答状态。
- 主操作跳转 `/questions` 或在当前 Tab 直接回答。

#### 评论

评论支持 Markdown 与嵌套回复。

结构：

- 评论输入框固定在评论列表上方。
- 顶层评论按时间升序。
- 回复缩进 24px，最多展示两层，继续回复仍归到该线程。
- 作者可编辑/删除自己的评论。
- 软删除且有子回复时保留“此评论已删除”占位。

#### 测试用例

列出关联 TestCase：

- 名称。
- 审核状态。
- 生成状态。
- 最近运行结果。
- 覆盖率。
- 操作：查看、运行。

入口：新建测试用例，进入测试用例创建抽屉。

#### 历史

显示 Issue 字段变更历史摘要，完整设计见 `07-AI问询执行日志与历史.md`。

### PR 与 Git 摘要

右侧栏底部展示：

- gitBranch。
- gitCommitSha。
- gitPRUrl。
- PR 状态。

无 PR 时根据状态显示创建入口。

### 状态冲突处理

如果用户操作时后端返回状态冲突：

- 弹出轻提示：当前状态已更新。
- 刷新 Issue 详情。
- 在右侧栏显示新状态与可用动作。

### 交互补充

- Issue 行点击标题/摘要区域进入详情；点击行内动作只触发该动作，不进入详情。
- 状态 Tabs 切换时只刷新下方行列表，保留工具栏筛选条件。
- Issue 详情页执行主动作后，右侧工作流栏立即进入 loading，主内容区不跳转，成功后再刷新状态。
- 执行、暂停、恢复、取消等动作完成后，自动在详情页顶部显示本次动作的时间和操作者。
- 评论提交失败时保留输入内容，并在评论框上方显示错误条。
- 若 Issue 有 pending 问询，详情页默认将 AI 问询 Tab 标为高亮，但不强制切换用户当前 Tab。
