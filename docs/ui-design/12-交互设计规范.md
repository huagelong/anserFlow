# 交互设计规范

## 适用范围

本文补充 `docs/ui-design` 中各页面的交互行为，覆盖用户动作、系统反馈、状态变化、失败恢复与响应式操作方式。本文不新增接口、不新增业务能力，只把已有页面、API 与状态机转化为可实施的交互规则。

## 交互原则

- 用户每次点击后都要知道系统是否接收、正在处理什么、下一步能做什么。
- 状态驱动动作：按钮只展示当前状态允许的动作；无法执行的动作要说明原因。
- 高风险操作慢一点，普通操作快一点：删除、吊销、关闭、驳回、取消执行必须确认；筛选、切换、展开不打断用户。
- 长任务不假装即时完成：执行、导入、生成、同步、测试运行都展示队列、进行中和最终结果。
- 失败后保留上下文：表单内容、筛选条件、日志和拆解结果不因错误丢失。

## 全局反馈模型

| 场景 | 反馈位置 | 反馈方式 |
|------|----------|----------|
| 普通保存成功 | 页面右上角 | 成功 toast，2.4 秒后自动消失。 |
| 表单校验失败 | 字段下方 | 红色错误文案，聚焦到第一个错误字段。 |
| 后端业务错误 | 当前区域顶部 | 错误条 + 重试或修改入口。 |
| 状态冲突 | 当前页面右侧栏或弹窗内 | 显示最新状态，要求用户刷新或按新状态继续。 |
| 长任务已创建 | 当前弹窗或抽屉 | 切换为任务状态视图，不立即关闭。 |
| 危险操作完成 | 当前页面 | 成功 toast + 列表/详情同步刷新。 |
| WebSocket 断开 | 日志面板顶部 | 黄色连接条，保留已有日志并自动重连。 |

Toast 只用于低风险结果反馈；需要用户阅读或修正的问题用错误条、弹窗或抽屉内提示。

## 按钮与动作状态

按钮状态：

- 默认：可点击。
- Hover：背景轻微加深，不改变尺寸。
- Loading：保留原宽度，显示 spinner，禁止重复提交。
- Disabled：保留位置，tooltip 说明原因，如“当前状态不允许执行”。
- Danger：红色文字或红色按钮，仅用于不可逆或高风险动作。

动作优先级：

1. 页面主按钮：当前最推荐动作。
2. 行内主操作：当前对象下一步动作。
3. 更多菜单：低频、危险或管理员动作。

同一区域最多出现一个主按钮。若同一状态有多个可行动作，主按钮只放最可能的下一步，其余进入次按钮或更多菜单。

## 表单交互

### 输入与校验

- 必填字段失焦后校验；提交时全量校验。
- Git URL、API endpoint、slug 等结构化字段输入时只做格式提示，真正连通性校验由用户点击“校验”或“测试连接”触发。
- Markdown 大文本不做实时强校验，只在提交时检查必填和长度。
- 密钥、Token、SSH 私钥默认隐藏；“显示 10 秒”后自动恢复隐藏。

### 脏数据保护

以下场景离开页面或关闭抽屉时必须提示：

- 项目创建流程未完成。
- Issue、需求、测试用例、Skill 编辑器有未保存内容。
- 拆解结果已被用户手动调整但未确认创建 Issue。
- PR 创建弹窗内用户编辑过标题或描述。

提示文案结构：

```
有未保存的修改
离开后当前输入不会保存。
[继续编辑] [放弃修改]
```

### 保存成功后的落点

- 新建对象成功：关闭抽屉，列表插入新项，并短暂高亮 1.6 秒。
- 编辑对象成功：留在当前页面，更新摘要和更新时间。
- 分步创建成功：进入完成页，不自动跳走。

## 列表与筛选

- 搜索输入使用 300ms 防抖。
- 筛选条件变更后列表局部刷新，不重置滚动到页面顶部。
- 清空筛选按钮仅在存在筛选条件时显示。
- 分页或游标加载时保留已选筛选条件。
- 表格行点击进入详情；行内按钮点击不触发行跳转。
- 删除或吊销成功后，当前行从列表中移除，并显示成功反馈。

## 抽屉、弹窗与确认

### 抽屉

抽屉用于不中断当前列表上下文的编辑、创建、查看详情。

- 打开时焦点进入第一个可编辑字段。
- Esc 可关闭无修改抽屉；有未保存修改时触发脏数据保护。
- 保存成功后关闭，失败时停留在抽屉内。

### 弹窗

弹窗用于确认、危险操作和短流程。

- 危险确认默认焦点不落在危险按钮上。
- 危险弹窗必须展示对象名称、影响范围和不可逆说明。
- 删除项目需要输入项目 slug；吊销 key、删除凭证、吊销设备需要确认对象名称。

## 状态机交互

### Issue 状态动作

Issue 的所有动作来自状态机和 API。交互层只做三件事：

- 根据当前状态展示允许动作。
- 触发前进行必要确认。
- 成功后刷新 Issue 状态、日志摘要、PR/Git 摘要和审计摘要。

动作触发后：

- `approve`：右侧状态栏从 draft 更新为 approved，执行计划区显示可生成计划。
- `execute`：状态进入 queued，展示队列信息；进入 running 后自动打开日志 Tab 的连接状态。
- `pause` / `resume`：按钮 loading 到状态更新，日志面板保留当前位置。
- `answer` / `skip_question`：问询卡片变为处理中，成功后状态回到 running。
- `retry`：状态回到 approved，保留上次失败日志入口。
- `create_pr`：成功后显示 PR URL 和编号。
- `close` / `reopen`：成功后右侧栏刷新可用动作。

### TestCase 状态动作

测试用例有审核和生成两个节奏：

- draft：用户可编辑描述和脚本。
- pending_review：表单只读，等待管理员审核。
- approved：主区域显示“等待 AI 生成测试代码”，不要求用户重复点击生成。
- generating：展示生成状态，禁止编辑描述。
- generated：显示生成代码，可运行测试。
- passed / failed：展示结果；failed 提供编辑描述和重新运行入口。
- rejected：展示驳回原因，允许编辑后重新提交。

## 长任务交互

适用任务：

- Git Issue 导入。
- AI 执行。
- AI 拆解需求。
- 测试代码生成。
- 测试运行。
- Skills 同步。
- Git URL 校验和 AI 工具测试连接。

统一展示：

| 阶段 | UI 表现 |
|------|---------|
| 创建中 | 主按钮 loading，禁止重复点击。 |
| queued | 显示任务已创建、排队中。 |
| running | 显示进度步骤或实时日志，不显示虚假百分比。 |
| succeeded | 展示结果摘要和下一步入口。 |
| failed | 展示错误详情、重试入口和返回编辑入口。 |

长任务弹窗关闭后，相关页面要保留状态入口，例如导入 Git Issues 后在 Issue 看板工具栏显示最近一次导入状态。

## 实时与通知

实时信息只用于文档中已有的执行日志和状态通知。

- 日志面板连接成功后显示绿色小点和“实时连接”。
- 断开后显示“实时连接已断开，正在重连...”，不清空日志。
- 通知中心只收纳需要用户回来的事件：待回答问询、执行失败、测试失败、导入完成、登录异常。
- 点击通知进入对应对象详情，若对象已不存在，展示错误条和返回入口。

## 拖拽交互

Issue 看板拖拽是状态动作的可视化入口，不是自由移动。

- 拖起卡片时原位置保留占位，高度不跳变。
- 可投放列显示主色边框，不可投放列置灰。
- 投放到可用列后弹出确认，说明将触发的状态事件。
- 投放到不可用列时卡片回原位，并显示原因。
- 移动端不使用拖拽，改用卡片更多菜单选择可用动作。

## 移动端交互

- 侧边栏使用抽屉。
- 看板改为状态 Tabs，每个 Tab 内为卡片列表。
- 详情页右侧工作流栏折叠为顶部“状态与操作”面板。
- 表格改为卡片列表，保留标题、状态、优先级和一个主操作。
- 长文本编辑器使用编辑/预览 Tabs，不做左右双栏。

## 可访问性与焦点

- 弹窗打开后焦点进入标题或第一个字段；关闭后焦点回到触发按钮。
- 所有图标按钮有 tooltip 和 `aria-label`。
- 状态变化不只靠颜色表达，必须有文字徽章。
- 表单错误和字段关联，屏幕阅读器能读到错误。
- 看板拖拽必须提供菜单式替代动作。

## 交互验收清单

- 所有主动作点击后都有 loading、成功或失败反馈。
- 所有危险动作都有确认，且确认文案包含对象名称。
- 状态动作只在对应状态显示。
- 长任务失败后保留用户输入和结果上下文。
- 列表筛选不会清空当前页面上下文。
- 移动端可完成查看、审批、回答问询、运行测试等轻操作。
