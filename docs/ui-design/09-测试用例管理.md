# 测试用例管理页面设计

覆盖页面：

- `/projects/:pid/test-cases`：TestCases.vue
- `/projects/:pid/test-cases/:tid`：TestCaseDetail.vue
- `/projects/:pid/test-cases/:tid/history`：TestCaseHistory.vue
- IssueDetail.vue 中的测试用例区域

## 测试用例列表

### 页面目标

集中查看项目内测试用例的审核、生成和运行状态，快速定位失败测试。

### 顶部区域

标题栏：

- 标题：测试用例。
- 副标题：展示项目默认测试框架和覆盖率目标。

测试用例通常从 Issue 详情创建；列表页提供只读扫描和跳转，不单独发明无 Issue 的测试创建入口。

工具栏：

- 搜索：测试名称、关联 Issue。
- 状态筛选：draft / pending_review / approved / generating / generated / passed / failed / rejected。
- 语言筛选。
- 框架筛选。
- 结果筛选：passed / failed。

### 表格列

| 列 | 宽度 | 内容 |
|----|------|------|
| 测试用例 | 320px | 名称、描述摘要。 |
| 关联 Issue | 280px | 标题、状态。 |
| 状态 | 140px | 生命周期状态。 |
| 审核 | 140px | reviewStatus、审核人。 |
| 技术 | 160px | language / framework。 |
| 最近结果 | 160px | passed / failed、覆盖率。 |
| 更新时间 | 160px | 相对时间。 |
| 操作 | 120px | 查看、运行。 |

失败行使用左侧 3px 危险色，但不改变整行可读性。

## 新建测试用例

入口在 Issue 详情的“测试用例” Tab。

抽屉宽 640px。

字段：

- 名称，必填。
- 描述，Markdown，多行，用户定义测试场景。
- language。
- framework。
- testScript，可选，使用代码编辑器。

说明：

- 用户只写测试描述，不直接要求填写生成后的 testCode。
- testScript 只覆盖执行脚本，优先级高于 framework 默认映射。

保存后状态为 `draft`。

## 测试用例详情

### 页面目标

让用户完成测试描述审核、AI 生成测试代码、运行测试和查看结果。

### 布局

桌面两栏：

- 左侧：描述、生成代码、结果、历史。
- 右侧：状态机、元信息、操作。

右侧栏 340px。

### 状态动作

| 当前状态 | 主动作 | 次动作 |
|----------|--------|--------|
| draft | 提交审核 | 编辑、删除 |
| pending_review | 审核通过 | 驳回 |
| approved | 等待自动生成 | 重新生成、编辑 |
| generating | 查看生成日志 | - |
| generated | 运行测试 | 重新生成 |
| passed | 查看结果 | 重新运行 |
| failed | 查看失败详情 | 重新运行、编辑描述 |
| rejected | 编辑后重新提交 | 删除 |

审核通过后自动触发 AI 生成时，主区域显示生成状态，不要求用户再次点击生成。

### 描述区

Markdown 展示用户定义的测试场景。

编辑规则：

- draft / rejected / failed 可以编辑描述。
- generated / passed 状态编辑前提示会影响后续生成结果。

### 生成代码区

当 `testCode` 存在时显示 Monaco 只读编辑器：

- 字号 13px。
- 语言根据 language 映射。
- 顶部显示 generatedBy。
- 提供复制按钮。

无代码时：

- approved 或 generating 状态显示生成进度。
- draft / pending_review 显示审核提示。

### testScript 区

如果用户配置了 testScript：

- 展示 install、command、timeout。
- 使用代码块或 JSON 折叠面板。
- 标记“自定义脚本优先”。

未配置时：

- 展示 framework 默认命令摘要。

## 测试结果卡片

结果区展示：

- status：passed / failed。
- duration。
- coverage。
- total / passed / failed / skipped。
- errors。
- output 摘要。

布局：

- 顶部 4 个小指标卡。
- 下方错误列表。
- 最后是原始输出折叠面板。

失败详情：

- 每个错误显示文件、行号、消息。
- 支持复制错误。
- 不自动创建 Issue，除非用户通过已定义 Issue 流程手动记录。

## TestCase 历史页

### 页面目标

查看测试描述、脚本、生成代码和结果变更历史，支持版本 diff 与回滚。

列表列：

- action。
- 操作人。
- 来源。
- 时间。
- 摘要。
- 查看 diff。

Diff 页面：

- 左右双栏。
- testCode 使用代码 diff。
- result 使用 JSON diff。

回滚：

- 仅回滚测试描述、testScript 等可编辑内容。
- 不回滚审计日志和历史记录自身。

## 空态与错误态

- 无测试用例：提示“从 Issue 详情创建测试用例”。
- 生成失败：展示错误摘要，允许重新生成。
- 运行失败：保留失败输出，允许重新运行。
- 权限不足：审核按钮隐藏或禁用。
