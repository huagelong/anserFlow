# 登录、组织与设备页面设计

覆盖页面：

- `/login`：Login.vue
- `/org/:slug`：Organization.vue
- `/org/:slug/members/:uid/edit`：OrgMemberEdit.vue
- `/org/:slug/users/new`：OrgUserCreate.vue
- `/org/:slug/devices`：DeviceManagement.vue

## 登录页

### 目标

让用户快速进入已有组织，或完成注册并创建/选择组织。页面只承载认证、注册、组织选择和语言切换，不展示营销内容。

### 布局

桌面端使用居中登录面板：

- 页面背景：`#F5F7FA`。
- 面板宽度：420px。
- 面板 padding：32px。
- 面板圆角：4px，边框 `#DCDFE6`。
- 面板顶部使用品牌大号版标识，不单独显示纯文本产品名。
- 登录页标识规格：图形 36x36px，字标 32px/700，图形与字标间距 12px。
- 字标分色：`flow` 使用主文字色，`code` 使用主色。

移动端：

- 面板宽度 `calc(100vw - 32px)`。
- padding 24px。
- 顶部品牌标识与语言切换保留，标识可按宽度等比缩小，但不得拆散图形与字标。

### 登录表单

字段：

| 字段 | 控件 | 规格 |
|------|------|------|
| 账号 | 输入框 | 支持用户名或邮箱，32px 高。 |
| 密码 | 密码输入框 | 支持显示/隐藏。 |
| 语言 | 下拉或分段控件 | 中文 / English。 |

动作：

- 主按钮：登录。
- 次操作：注册账号。
- 成功后如用户只有一个组织，直接进入项目仪表盘；多个组织进入组织选择。

状态：

- 登录中：按钮 loading，表单不可重复提交。
- 失败：表单顶部错误条，显示后端错误信息。
- token 过期：跳转登录页并保留来源路径，登录后返回。

### 注册表单

字段：

- 用户名。
- 邮箱。
- 密码。
- 默认语言。

注册成功后进入组织创建/选择，不自动创建项目。

## 组织选择

### 触发

登录后用户属于多个组织时显示；用户无组织时引导创建组织。

### 布局

同登录面板，面板宽 520px。

组织列表项：

- 高度 64px。
- 左侧组织头像或首字母。
- 中间组织名称、slug、成员角色。
- 右侧进入箭头。

空态：

- 标题：暂无组织。
- 主操作：创建组织。

## 组织管理页

### 页面目标

管理组织基本信息、成员和组织下项目入口。设备管理独立页面，但在组织页保留入口。

### 页面结构

```
标题区：组织名称 / slug / 当前用户角色 / 编辑组织
Tabs：成员 / 项目 / 设置
```

### 成员 Tab

表格列：

| 列 | 宽度 | 说明 |
|----|------|------|
| 成员 | 240px | 头像、昵称、邮箱。 |
| 角色 | 120px | owner / admin / member。 |
| 状态 | 120px | active / invited。 |
| 加入时间 | 160px | 日期。 |
| 操作 | 160px | 修改角色、移除。 |

操作：

- 邀请成员：弹窗，输入邮箱和角色。
- 编辑成员信息：管理员入口，跳转 `/org/:slug/members/:uid/edit`，维护成员资料与组织角色。
- 手动添加用户：管理员入口，跳转 `/org/:slug/users/new`，直接创建账号并加入当前组织。
- 修改角色：下拉选择，保存后表格行内更新。
- 移除成员：危险确认弹窗。

权限：

- 非管理员只读成员列表。
- 不能移除自己作为唯一 owner 的组织。

### 编辑成员信息页

### 页面目标

用于组织管理员维护已有成员资料，统一处理邮箱、语言偏好、头像地址以及组织角色调整。页面强调“编辑资料”与“修改密码”分离，不在此页直接重置成员密码。

### 页面布局

- 保留组织管理页的顶栏与左侧导航，仍处于“组织管理”功能分区。
- 页面头部展示成员标题、管理员权限标识，以及“返回成员列表”按钮。
- 内容区采用左右两栏：左侧为成员资料与权限编辑，右侧为规则说明、最近活动和变更预览。

### 表单字段

成员资料：
- 用户名，只读。
- 邮箱，可编辑。
- 语言偏好，可编辑。
- 头像地址，可编辑。
- 加入时间，只读。
- 最近登录，只读。

组织权限：
- 组织角色：`owner / admin / member` 中按权限显示可选项。
- 当前状态：只读展示 `active / invited` 等成员状态。

### 操作

- 主按钮：保存成员信息。
- 次按钮：重置改动。
- 返回动作：取消或返回成员列表。

### 权限与约束

- 仅组织管理员可见，普通成员不可进入。
- 用户名在账号创建后冻结，管理员和成员本人都不能修改。
- 角色修改只影响当前组织，不影响该用户在其他组织中的身份。
- 若成员是当前组织唯一 owner，则禁止降级 owner 权限。
- 保存成功后需刷新成员列表，并写入组织级审计日志。

### 手动添加用户页

### 页面目标

用于管理员在后台直接创建组织成员账号，适合内部员工开通、测试账号准备和线下审批后录入。与“邀请成员”不同，此页允许一次性设置初始密码、账号状态和首次登录策略。

### 页面布局

- 保留组织管理页的顶栏与左侧导航，仍处于“组织管理”功能分区。
- 内容区分为左右两栏：左侧为创建表单，右侧为规则说明、操作流程和最近手动创建记录。
- 页面头部展示标题“手动添加用户”、管理员权限标识，以及“返回成员列表”次按钮。

### 表单字段

基础资料：
- 用户名。
- 邮箱。
- 姓名 / 昵称。
- 手机号（选填）。
- 初始密码。
- 确认密码。

组织与安全：
- 组织角色：`owner / admin / member` 中按权限显示可选项。
- 账号状态：`active` 或其他后端允许的初始状态。
- 发送初始化邮件：布尔开关。
- 首次登录强制修改密码：布尔开关，默认开启。

补充信息：
- 备注 / 来源说明：用于记录审批单号、部门、测试用途等审计信息。

### 操作

- 主按钮：创建用户。
- 次按钮：创建并继续添加。
- 返回动作：取消或返回成员列表，不修改已存在成员数据。

### 权限与约束

- 仅组织管理员可见，普通成员不可进入。
- 用户名与邮箱必须满足唯一性校验，错误在字段下方或表单顶部反馈。
- 若关闭“发送初始化邮件”，页面仍需明确提示管理员线下安全交付密码。
- 创建成功后用户直接加入当前组织，并出现在成员列表中。

### 项目 Tab

展示组织内项目列表：

- 项目名、slug、Git provider、默认分支、Issue 数、最近更新时间。
- 主操作：新建项目，跳转 `/projects/new`。

### 设置 Tab

字段：

- 组织名称。
- slug，只允许在创建后按后端规则更新。
- 组织级设置 JSON 的可视化摘要，不暴露未定义配置项。

保存动作：

- 使用 `PUT /api/v1/orgs/:slug`。
- 保存成功显示 toast，不离开页面。

## 设备管理页

### 页面目标

展示当前用户已登录设备，支持远程吊销和保留当前设备。

### 布局

标题栏：

- 标题：已登录设备。
- 副标题：展示当前账号的活跃设备。
- 右侧危险次按钮：注销其他设备。

设备卡片或表格：

| 列 | 说明 |
|----|------|
| 设备 | 设备名、OS、架构。 |
| 来源 | web / cli。 |
| IP | 最近登录 IP。 |
| 最近活跃 | 时间。 |
| 当前设备 | 当前设备显示绿色徽章。 |
| 操作 | 吊销。 |

交互：

- 当前设备不可单独吊销，只能通过退出登录。
- 吊销其他设备弹窗显示设备名、来源、最近活跃时间。
- 批量“注销其他设备”保留当前设备。

安全细节：

- 新设备登录通知从通知中心进入本页。
- 登录异常仍在审计日志页展示，本页只管理设备。

## 交互补充

- 登录失败后保留账号输入，清空密码并聚焦密码框。
- 多组织用户登录后进入组织选择；选择组织时列表项进入 loading，避免重复点击。
- 创建组织成功后自动切换到新组织，并进入项目列表。
- 邀请成员弹窗提交成功后保留弹窗关闭前的角色选择，便于连续邀请同角色成员。
- 吊销设备成功后该设备行从列表中移除；如果吊销的是其他设备，当前页面不跳转。
- “注销其他设备”成功后列表只保留当前设备，并显示成功 toast。
