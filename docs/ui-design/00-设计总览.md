# flowcode Web 控制台 UI 设计总览

## 设计边界

本设计文档只覆盖 `docs` 中已定义的 Web 控制台功能，不扩展未出现的产品能力。CLI 与 REST API 不单独设计界面，只作为 Web 页面里的操作、状态、配置与反馈来源。

主要来源文档：

- `docs/01-项目概述.md`：产品定位、目标用户、核心功能矩阵、接入形态。
- `docs/02-系统架构.md`：前端路由、页面清单、组件目录、导航结构。
- `docs/03-数据模型.md`：实体字段、状态、配置项。
- `docs/04-API设计.md`：页面可执行操作和接口边界。
- `docs/05-工作流引擎.md`：Issue 与 TestCase 状态机。
- `docs/07-AI编程工具集成.md`：AI 执行、主动询问、需求拆解、测试生成。
- `docs/08-Git集成方案.md`：Git 凭证、PR、手动导入 Issue。
- `docs/09-监控与反馈闭环.md`：监控面板、事件转 Issue、反馈闭环。
- `docs/10-部署与运维.md`：健康检查、日志、运维指标。
- `docs/11-测试策略.md`：测试链路与质量验证。

## 设计假设

- 默认目标用户是独立开发者、小团队管理员、CLI 重度用户转到 Web 查看状态的人。
- 控制台是工作型 SaaS 工具，信息密度应高于营销页面，优先保证扫描、筛选、决策和批量处理效率。
- 技术栈按文档采用 Vue 3 + TypeScript + Vite + Element Plus + Pinia + Vue Router。
- 设计以桌面端为主，兼顾平板与移动端查看、审批、回复 AI 问询等轻操作。
- 所有危险操作必须二次确认：删除项目、删除凭证、吊销 API Key、吊销设备、关闭/重开 Issue、回滚历史、删除 Skill。

## 核心工作流

### 1. 首次使用

登录或注册 → 选择/创建组织 → 进入项目列表 → 新建项目 → 校验 Git 地址 → 选择默认分支和凭证 → 进入仪表盘。

### 2. 需求到执行

创建需求 → 审核为 `approved` → 调用 AI 拆解 → 用户确认子 Issue → Issue 进入 `draft` → 管理员审核 → 生成计划 → 手动或自动入队执行 → 实时日志 → 完成后创建 PR。

### 3. Issue 处理

看板筛选问题 → 打开详情 → 查看 AI 分类、分析和计划 → 审批/驳回/执行/暂停/取消/重试 → 评论协作 → 关联测试用例 → 追踪 PR 与部署状态 → 关闭或重开。

### 4. AI 主动询问

执行中 AI 输出 `[FLOWCODE_Q]` → Issue 状态进入 `waiting_input` → 页面与全局通知提示 → 用户选择建议项或手写回答 → AI 继续执行；用户也可跳过，让 AI 自行决策。

### 5. 监控反馈闭环

线上事件上报 → 监控面板聚合与去重 → 阈值触发自动 Issue 或用户手动转 Issue → 进入 Issue 工作流 → 修复、PR、部署后追溯事件是否消失。

## 体验原则

- 页面先展示“我现在该看什么、该点什么”，其次才展示完整数据。
- 状态与动作必须贴近：卡片、详情页、表格行都显示下一步可用动作，不展示当前状态不允许的按钮。
- 长流程使用分步条或右侧状态栏，避免用户在多个页面间猜当前进度。
- AI 输出内容要可读、可追踪、可覆盖：分析、分类、计划、日志和问询都要标明来源与时间。
- 表格负责扫描，详情页负责决策，抽屉/弹窗负责轻量编辑。

交互细节统一见 `12-交互设计规范.md`，各页面文档只保留与本页强相关的特殊交互。

## 断点与响应式

| 断点 | 宽度 | 布局策略 |
|------|------|----------|
| Mobile | 360-767px | 单列；侧边栏抽屉；看板变为状态 Tabs；详情右侧栏折叠到顶部摘要。 |
| Tablet | 768-1199px | 侧边栏可折叠；表格保留核心列；详情页双栏改 60/40。 |
| Desktop | 1200-1599px | 固定侧边栏 + 内容区；看板 5-7 列横向滚动。 |
| Wide | 1600px+ | 内容最大宽度 1440px；看板和日志允许扩展到全宽。 |

## 验收标准

- 每个文档中列出的页面都能追溯到 `docs/02-系统架构.md` 的路由。
- 每个页面的按钮动作都能追溯到 `docs/04-API设计.md` 的接口或 `docs/05-工作流引擎.md` 的状态动作。
- 所有状态色、优先级、分类、空态、错误态、加载态在视觉规范中有统一定义。
- 所有长任务、危险操作、状态冲突和移动端轻操作都能追溯到 `12-交互设计规范.md`。
- 设计不包含文档未定义的自动同步、定时 Git 导入、聊天机器人、账单计费等额外能力。
