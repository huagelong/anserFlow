# 视觉与组件规范

## 设计语气

flowcode 是面向开发者和小团队的工作型控制台。视觉应清晰、克制、可信，避免营销化大图和装饰性元素。核心体验是让用户快速判断：当前项目是否健康、哪些 Issue 卡住、AI 在等什么、下一步该执行什么。

## 字体

| 类型 | 字体栈 | 用途 |
|------|--------|------|
| UI 文本 | `Inter`, `Noto Sans SC`, `PingFang SC`, `Microsoft YaHei`, sans-serif | 默认界面文字。 |
| 代码/日志 | `JetBrains Mono`, `SFMono-Regular`, `Consolas`, monospace | 终端、代码、diff、配置。 |

字号：

| Token | 大小 | 行高 | 用途 |
|-------|------|------|------|
| `text-xs` | 12px | 18px | 徽章、时间、表格辅助。 |
| `text-sm` | 13px | 20px | 表格、表单说明。 |
| `text-base` | 14px | 22px | 默认正文。 |
| `text-md` | 16px | 24px | 页面小标题、表单标题。 |
| `text-lg` | 18px | 28px | 区块标题。 |
| `text-xl` | 20px | 30px | 页面标题。 |
| `text-2xl` | 24px | 34px | 仪表盘关键标题。 |
| `text-3xl` | 32px | 40px | 登录页产品名。 |

字重：

- 常规：400
- 表格标题、按钮：500
- 页面标题、关键数字：600
- 只在少数关键数字使用 700

## 尺寸与间距

基础单位为 4px。

| Token | 值 | 用途 |
|-------|----|------|
| `space-1` | 4px | 图标与文字间距。 |
| `space-2` | 8px | 表单项内部、徽章组。 |
| `space-3` | 12px | 卡片内紧凑间距。 |
| `space-4` | 16px | 默认区块 padding。 |
| `space-5` | 20px | 表格工具栏。 |
| `space-6` | 24px | 页面区块间距。 |
| `space-8` | 32px | 大区块、详情页主间距。 |

圆角：

- 输入框、按钮、徽章：6px。
- 卡片、弹窗、抽屉：8px。
- 头像：50%。
- 不使用超过 8px 的卡片圆角。

阴影：

- 默认不用阴影，依靠背景和边框分层。
- 弹窗/下拉：`0 12px 32px rgba(15, 23, 42, 0.12)`。
- 拖拽卡片：`0 8px 20px rgba(15, 23, 42, 0.10)`。

## 通用组件

### 按钮

- 主按钮：用于当前页面最主要动作，如“新建 Issue”“确认拆解”“执行”。
- 次按钮：用于普通操作，如“编辑”“重新生成计划”。
- 危险按钮：仅用于删除、吊销、取消执行等。
- 图标按钮尺寸：32px，高频工具栏图标使用 tooltip。
- 按钮内文字不换行；在 360px 宽度下改为图标按钮或菜单项。

### 表单

- 标签宽度：普通表单 120px；窄弹窗 96px。
- 输入框高度：32px；大文本域最小 128px。
- 必填项在标签后显示红色 `*`。
- 校验信息显示在控件下方，字号 12px，颜色 `#DC2626`。
- 密钥、Token、SSH 私钥默认隐藏，提供“显示 10 秒”动作。

### 表格

- 工具栏高度 48px，左侧筛选，右侧主操作。
- 列过多时保留首列、状态列、操作列，其他进入详情抽屉或横向滚动。
- 行 hover 背景 `#F9FAFB`。
- 批量操作仅在用户勾选行后出现。

### 卡片

- 卡片用于重复项：项目、Issue、PR、测试结果、监控事件。
- 卡片边框 `1px solid #E5E7EB`，背景 `#FFFFFF`。
- 选中卡片边框 `#2563EB`，背景 `#EFF6FF`。
- 卡片标题最多两行，超出省略；描述最多三行。

### 状态徽章

- 高度 22px，左右 padding 8px，圆角 999px。
- 文本 12px/18px，字重 500。
- 状态、优先级、分类、来源都使用徽章，但同一区域最多并列 4 个，超出折叠菜单。

### Tabs

- 详情页使用横向 Tabs：概览、计划、日志、评论、测试、历史。
- 移动端 Tabs 可横向滚动，不压缩文字到不可读。
- 当前 Tab 下边框使用主色 2px。

### 抽屉与弹窗

- 抽屉用于编辑、筛选、配置、导入结果。
- 弹窗用于确认、轻量创建和危险操作。
- 危险确认弹窗必须展示操作对象名称，要求用户点击明确的危险按钮，不使用默认回车提交。

### 编辑器

- Markdown 编辑器默认双栏：左编辑、右预览；窄屏改 Tabs。
- Monaco 用于 `testScript`、diff 和代码内容，默认字号 13px。
- 代码块背景 `#111827`，文字 `#E5E7EB`。

### 终端日志

- 使用深色终端背景 `#111827`。
- 成功日志 `#86EFAC`，警告 `#FCD34D`，错误 `#FCA5A5`，普通文本 `#D1D5DB`。
- 顶部固定工具栏：连接状态、自动滚动开关、筛选级别、复制、查看完整 stdout。

### 图表

- 趋势图使用青色主线 `#0891B2`，错误叠加危险色 `#DC2626`。
- 图表必须有明确坐标轴、tooltip 和空态。
- 不使用装饰性渐变背景。

## 空态、加载态、错误态

| 场景 | 展示 |
|------|------|
| 空列表 | 页面内空态，图标 40px，标题 16px，说明 13px，给出一个主操作。 |
| 首次加载 | 骨架屏，保持最终布局尺寸，避免跳动。 |
| 局部刷新 | 表格右上角小 spinner，不遮挡已加载内容。 |
| 网络错误 | 行内错误条，提供“重试”。 |
| 权限不足 | 说明缺少权限，隐藏不可用主操作，保留只读内容。 |
| 状态冲突 | 显示当前最新状态和建议下一步，如“此 Issue 已进入 running，请刷新后操作”。 |

## 可访问性

- 普通正文对比度不低于 4.5:1。
- 所有图标按钮提供 `aria-label` 与 tooltip。
- 表单错误通过 `aria-describedby` 关联。
- 看板拖拽提供键盘替代：卡片操作菜单里可选择允许的状态动作。
- 颜色不作为唯一信息来源，状态徽章必须有文本。
