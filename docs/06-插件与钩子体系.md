# 06 - 插件与钩子体系

## 6.1 设计目标

flowcode 的插件体系追求 **零侵入扩展**。AI 编程工具和 Git 平台不硬编码在核心逻辑中，而是通过标准接口接入。新工具/平台的接入只需实现对应接口，无需修改核心代码。

## 6.3 AI Tool Adapter 接口规范

每种 AI 编程工具需实现 `AIToolAdapter` 接口：

```go
// internal/adapter/ai_tool/adapter.go

// AIToolConfig 工具元信息
type AIToolConfig struct {
    Name        string // 工具唯一标识
    DisplayName string // 显示名称
    Description string // 描述
    Icon        string // 图标
}

// ExecuteContext 执行上下文
type ExecuteContext struct {
    WorkspaceDir string            // 工作目录
    Prompt       string            // Issue 描述 (自然语言指令)
    ProjectCtx   ProjectContext    // 项目上下文
    Config       map[string]any    // 工具专属配置
    OnLog        func(LogEntry)    // 实时推送日志
    OnProgress   func(int, string) // 进度更新
    Ctx          context.Context   // 取消信号
}

type ProjectContext struct {
    RepoURL       string   // 仓库 URL
    Branch        string   // 当前分支
    LastCommitSHA string   // 上次 commit SHA (续跑时)
    TechStack     []string // 项目技术栈
    SkillContents []string // 加载的 Skills 内容
}

// ExecuteResult 执行结果
type ExecuteResult struct {
    Success      bool
    ExitCode     int
    FilesChanged []string
    Stdout       string
    Stderr       string
    Duration     int64 // ms
    Metadata     map[string]any
}

type LogEntry struct {
    Timestamp int64  // unix ms
    Level     string // info / warn / error / debug
    Content   string
    Step      string
}

// AIToolAdapter AI 工具适配器接口
type AIToolAdapter interface {
    Config() AIToolConfig
    ValidateConfig(cfg map[string]any) error
    Execute(ctx ExecuteContext) (*ExecuteResult, error)
    HealthCheck(cfg map[string]any) (available bool, latencyMs int64, err error)
    ParseCommitMessage(result *ExecuteResult) string
}
```

## 6.5 钩子系统 (Hook Registry)

### 架构

```
Workflow Engine ──emit──→ HookRegistry ──dispatch──→ [Hook1, Hook2, ...]
                              │
                              ├── in-process hooks (同步)
                              └── webhook hooks (异步 HTTP POST)
```

### 钩子注册

钩子在项目级别配置，存储于 `Project.settings.hooks`：


### 钩子执行流程

```go
// internal/service/hook_registry.go

type HookRegistry struct {
    logger   *slog.Logger
    hooks    map[string][]HookConfig
}

func (r *HookRegistry) Emit(ctx context.Context, event string, payload any) {
    hooks := r.hooks[event]
    for _, hook := range hooks {
        if hook.Type == "webhook" {
            go r.sendWebhook(ctx, hook, map[string]any{
                "event":     event,
                "payload":   payload,
                "timestamp": time.Now().UnixMilli(),
            })
        } else if hook.Type == "builtin" {
            r.executeBuiltin(ctx, hook, payload)
        }
    }
}

func (r *HookRegistry) sendWebhook(ctx context.Context, hook HookConfig, data any) {
    body, _ := json.Marshal(data)
    req, _ := http.NewRequestWithContext(ctx, "POST", hook.URL, bytes.NewReader(body))
    req.Header.Set("Content-Type", "application/json")
    req.Header.Set("X-FlowCode-Signature", r.sign(body, hook.Secret))
    req.Header.Set("X-FlowCode-Event", data.(map[string]any)["event"].(string))

    client := &http.Client{Timeout: 10 * time.Second}
    resp, err := client.Do(req)
    if err != nil {
        r.logger.Error("webhook failed", "url", hook.URL, "error", err)
        return
    }
    resp.Body.Close()
}
```

### Webhook 签名验证

采用 HMAC-SHA256 签名，防止伪造：

```
X-FlowCode-Signature: sha256=<HMAC(payload, secret)>
```

接收方可使用相同算法验证签名。

## 6.8 Skills 技能体系

### Skill 定义格式

每个 Skill 是一个 Markdown 文件，包含技能描述、使用指导和示例代码：

````markdown
---
name: go-best-practices
version: 1.2.0
category: language
description: Go 语言最佳实践编码规范
---

# Go Best Practices

## 编码规范

- 使用 `gofmt` 格式化所有代码
- 错误必须处理，不允许忽略 `err`
- 公开函数必须有 GoDoc 注释
- 使用 `context.Context` 传递请求上下文

## 项目结构

```
cmd/          # 入口
internal/     # 私有包
pkg/          # 可导出库
```

## 测试要求

- 使用 `testing` 标准库
- 表驱动测试优先
- 覆盖率目标 > 80%
````

### Skill 加载机制

```go
// internal/service/skill_loader.go

type SkillLoader struct {
    db     *gorm.DB
    fsPath string // Skills 文件存储路径
}

// LoadForIssue 根据 Issue 绑定的 skillIds 加载 Skill 内容
func (l *SkillLoader) LoadForIssue(ctx context.Context, issueID string) ([]string, error) {
    var issue model.Issue
    if err := l.db.First(&issue, "id = ?", issueID).Error; err != nil {
        return nil, err
    }

    var skillIDs []string
    json.Unmarshal([]byte(issue.SkillIDs), &skillIDs)

    var skills []model.Skill
    l.db.Find(&skills, "id IN ?", skillIDs)

    contents := make([]string, len(skills))
    for i, s := range skills {
        contents[i] = s.Content
    }
    return contents, nil
}
```

```go
// 在 AI 执行前，将 Skills 内容注入到 prompt 前缀
func buildPromptWithSkills(issue *model.Issue, skills []string) string {
    var sb strings.Builder
    
    // 先注入 Skills 作为系统指令
    sb.WriteString("## Active Skills (must follow)\n\n")
```

### Skill 市场与社区

- **组织级 Skills**：团队内部共享的编码规范、部署清单

### Skill 安全机制

Skills 的 Markdown 内容会被注入 AI prompt，因此必须防止恶意代码注入和供应链攻击。

#### Skill 内容安全审查

```go
// internal/service/skill_validator.go

type SkillValidator struct{}

func (v *SkillValidator) Validate(content string) error {
    // 1. 文件大小限制 (最大 64KB)
    if len(content) > 64*1024 {
        return fmt.Errorf("skill content exceeds 64KB limit")
    }

    // 2. 禁止危险指令 — 拒绝包含高风险命令的 Skill
    dangerous := []string{
        "rm -rf", "DROP TABLE", "eval(", "exec(",
        "curl | sh", "wget -O - |", "base64 -d |",
        "os.system", "subprocess.call", "child_process",
    }
    lower := strings.ToLower(content)
    for _, d := range dangerous {
        if strings.Contains(lower, d) {
            return fmt.Errorf("skill contains dangerous pattern: %s", d)
        }
    }

    // 3. Skill YAML 前置元数据必须合法
    if !strings.HasPrefix(strings.TrimSpace(content), "---") {
        return fmt.Errorf("skill must have YAML front matter")
    }

    return nil
}
```

#### Skill 签名验证

组织级 Skill 创建时自动签名，安装公共 Skill 时验证签名来源：

```go
// internal/service/skill_signer.go

type SkillSigner struct {
    orgPrivateKey *rsa.PrivateKey  // 组织 RSA 私钥
    publicKeys    map[string]*rsa.PublicKey // 受信任发布者公钥
}

// Sign 对 Skill 内容生成 Ed25519 签名
func (s *SkillSigner) Sign(content string) (string, error) {
    hash := sha256.Sum256([]byte(content))
    sig, err := rsa.SignPKCS1v15(rand.Reader, s.orgPrivateKey, crypto.SHA256, hash[:])
    if err != nil {
        return "", err
    }
    return base64.StdEncoding.EncodeToString(sig), nil
}

// Verify 验证 Skill 签名，确保内容未被篡改
func (s *SkillSigner) Verify(content, signature string, publisherID string) error {
    pubKey, ok := s.publicKeys[publisherID]
    if !ok {
        return fmt.Errorf("untrusted publisher: %s", publisherID)
    }
    hash := sha256.Sum256([]byte(content))
    sig, _ := base64.StdEncoding.DecodeString(signature)
    return rsa.VerifyPKCS1v15(pubKey, crypto.SHA256, hash[:], sig)
}
```

#### Skill 信任等级

| 等级 | 来源 | 安全审查 | 签名验证 | 自动注入 |
|------|------|:---:|:---:|:---:|
| **Trusted** | 组织内部创建 | — | ✅ 组织签名 | ✅ 自动 |
| **Verified** | 官方/认证发布者 | ✅ | ✅ 发布者签名 | ✅ |
| **Community** | 社区市场 | ❌ | ✅ 基本签名 | ⚠️ 用户确认 |
| **Untrusted** | URL 导入 | ❌ | ❌ | ❌ 需手动审核 |

```go
// 安装 Skill 时的安全流程
func (s *SkillService) InstallSkill(ctx context.Context, orgID, skillID string) error {
    skill, _ := s.repo.FindByID(skillID)

    switch skill.TrustLevel {
    case "untrusted":
        // 必须管理员手动审核
        if !s.isAdmin(ctx, orgID) {
            return fmt.Errorf("untrusted skills require admin approval")
        }
        // 内容审查
        if err := s.validator.Validate(skill.Content); err != nil {
            return fmt.Errorf("skill validation failed: %w", err)
        }
    case "community":
        // 用户需确认安装来源
        if err := s.validator.Validate(skill.Content); err != nil {
            return fmt.Errorf("skill validation failed: %w", err)
        }
        // 签名验证
        if err := s.signer.Verify(skill.Content, skill.Signature, skill.CreatorID); err != nil {
            return fmt.Errorf("skill signature invalid: %w", err)
        }
    case "verified":
        if err := s.signer.Verify(skill.Content, skill.Signature, skill.CreatorID); err != nil {
            return fmt.Errorf("skill signature invalid: %w", err)
        }
    case "trusted":
        // 组织内部 Skill，直接放行
    }

    return s.repo.Install(orgID, skillID)
}
```

#### Skill 注入 AI Prompt 时的防护

```go
// 注入前做沙箱化处理，防止 prompt injection
func sanitizeSkillForPrompt(skillContent string) string {
    var sb strings.Builder

    // 将 Skill 内容包裹在明确的边界标记中
    sb.WriteString("<skill_instructions>\n")
    sb.WriteString("The following are mandatory coding guidelines. Follow them strictly.\n\n")
    sb.WriteString(skillContent)
    sb.WriteString("\n</skill_instructions>\n\n")
    sb.WriteString("<user_task>\n")

    return sb.String()
}
```

关键防护措施：

| 措施 | 说明 |
|------|------|
| 内容审查 | 禁止 Skill 包含 shell 命令、代码执行语句、SQL 注入模式 |
| 大小限制 | 单 Skill 最大 64KB，防止资源耗尽 |
| 签名验证 | 公共 Skill 安装时验证 Ed25519 签名 |
| 信任等级 | 四级信任体系，低信任 Skill 需管理员审核 |
| Prompt 隔离 | Skill 内容用 XML 标签包裹，防止 prompt injection 越权 |
| 组织中隔离 | 安装的 Skill 仅在组织范围内生效，不跨组织泄漏 |

## 6.9 安全钩子中间件

所有插件执行前经过统一安全检查：

```go
// internal/service/hook_guard.go

type HookSecurityGuard struct {
    blockedURLs []string        // 禁止的目标 URL (内网地址等)
    maxWebhookBatch int         // 最大批次数
}

func (g *HookSecurityGuard) CheckWebhook(urlStr string) error {
    u, _ := url.Parse(urlStr)

    // 禁止向以下地址发送 webhook
    blocked := []string{"127.0.0.1", "localhost", "10.", "172.16.", "192.168.", "0.0.0.0"}
    for _, b := range blocked {
        if strings.Contains(u.Host, b) {
            return fmt.Errorf("webhook to internal network is blocked: %s", u.Host)
        }
    }

    // 仅允许 HTTPS (生产环境)
    if os.Getenv("GIN_MODE") == "release" && u.Scheme != "https" {
        return fmt.Errorf("webhook must use HTTPS in production")
    }

    return nil
}
```
