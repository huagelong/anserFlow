工具之间不依赖任何网络协议或 RPC，通过 **文件注入 + 子进程 CLI + 管道捕获** 三层操作系统级通道通讯：  ``` ┌──────────────────────────────────────────────────┐ │              flowcode Server (Go)                │ │                                                  │ │  ① 写入 .flowcode_prompt.md                      │ │  ② exec.CommandContext("opencode", "run", ...)   │ │  ③ stdoutPipe / stderrPipe 实时捕获              │ │  ④ cmd.Wait() → exitCode 判定结果               │ └────────────┬─────────────────────────────────────┘              │ Docker SDK ┌────────────┴─────────────────────────────────────┐ │            Docker 容器（沙盒）                     │ │                                                  │ │  /workspace/                                     │ │  ├── .flowcode_prompt.md   ← ① Prompt 注入       │ │  └── repo/                 ← AI 工具读写代码      │ │                                                  │ │  opencode CLI 子进程                             │ │  ├── stdin   (不使用)                            │ │  ├── stdout  ──→ ③ pipe → WebSocket 推前端       │ │  └── stderr  ──→ ③ pipe                         │ └──────────────────────────────────────────────────┘ ```  | 方向 | 通道 | 内容 | 说明 | |------|------|------|------| | flowcode → AI 工具 | **文件系统** | `.flowcode_prompt.md` | Issue 标题+描述+分析摘要+执行计划+Skills 指令，写入容器 `/workspace` | | flowcode → AI 工具 | **环境变量** | `OPENAI_API_KEY` / `ANTHROPIC_API_KEY` 等 | 通过 `cmd.Env` 注入，不落地文件 | | flowcode 启动 AI | **子进程** | `exec.CommandContext` | 在沙盒容器内以 CLI 方式启动 AI 工具 | | AI 工具 → flowcode | **stdout pipe** | 实时执行日志（JSON Lines） | 流式捕获 → `ctx.OnLog()` → WebSocket 推送前端 | | AI 工具 → flowcode | **stderr pipe** | 警告/错误信息 | 流式捕获，level=warn | | AI 工具 → flowcode | **退出码** | 0=成功, 非0=失败 | `cmd.Wait()` 获取 | | AI 工具 ↔ 代码仓库 | **文件系统** | `/workspace/repo` | AI 工具通过标准文件读写操作源代码 |  **分析/分类场景**（无需代码仓库）：  ```go // 直接通过 --input 传 prompt，stdout 取 JSON 结果 cmd := exec.CommandContext(ctx, "opencode", "run",     "--input", prompt,     "--model", "claude-haiku",     "--max-turns", "1",     "--output-format", "json", ) output, _ := cmd.CombinedOutput() // 解析 output 中的 JSON ```  > **为什么不走网络协议？** AI 工具均为 CLI 程序，子进程是最直接的调用方式。管道捕获 stdout/stderr 天然支持流式输出，无需实现 WebSocket client 端或 gRPC stream。Docker 容器提供进程级隔离和资源限制。
