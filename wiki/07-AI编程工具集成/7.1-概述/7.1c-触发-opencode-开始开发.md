一条命令：创建 Issue + 触发 opencode 执行 flowcode issue create "修复登录页面样式错乱" --priority p0 --tool opencode  # 指定 Skills + AI 配置 flowcode issue create "实现用户认证" -p p0 --tool claude-code \   --skill auth,testing \   --config '{"model":"claude-sonnet-4-20250514","maxTurns":60}'  # 实时查看执行日志 flowcode issue watch abc123 ```  CLI 自动完成 **创建** 并触发前置分析；审批通过后可先生成执行计划，真正执行前若缺失则由 Asynq Worker 自动补齐。  #### 方式 2：Web 控制台  ``` 看板页面   │   ├── 1. 点击「新建 Issue」 → 填写标题/描述 → 选择 AI 工具 → 提交   │      → Issue 状态: draft   │   ├── 2. 管理员点击「审批通过」   │      → Issue 状态: approved   │      → 可查看/生成执行计划；若直接执行则系统在执行前自动补齐   │   └── 3. 若项目开启 autoExecute → 自动入队执行          否则 → 管理员点击「执行」→ 手动入队               → Issue 状态: queued → running               → 底部面板实时显示执行日志流 ```  #### 方式 3：REST API  ```bash # 1. 创建 Issue curl -X POST https://flowcode.example.com/api/v1/projects/proj_1/issues \   -H "Authorization: Bearer $JWT_TOKEN" \   -H "Content-Type: application/json" \   -d '{     "title": "修复登录页面样式错乱",     "description": "登录按钮在 320px 宽度下溢出...",     "priority": "p0",     "aiTool": "opencode",     "skillIds": ["skill_css_fix"]   }'  # 2. 审核通过（可触发计划生成） curl -X POST https://flowcode.example.com/api/v1/issues/iss_abc123/approve \   -H "Authorization: Bearer $JWT_TOKEN"  # 3. 手动重生成计划（可选） curl -X POST https://flowcode.example.com/api/v1/issues/iss_abc123/plan \   -H "Authorization: Bearer $JWT_TOKEN"  # 4. 触发执行（若项目未开启 autoExecute） curl -X POST https://flowcode.example.com/api/v1/issues/iss_abc123/execute \   -H "Authorization: Bearer $JWT_TOKEN" ```  #### 底层执行链路  ``` Issue approve / plan / execute     │     ▼ afterEnqueue() → Asynq 投递 "issue:execute" 任务     │     ▼ Worker pickup → RestoreIssueFSM() → Transition("pickup")     │     ▼ ExecuteIssueHandler()     │     ├── ① 前置分析检查（必要时补做 analyze）     │     ├── ② needsCodeChange 检查（false → skip，跳过执行）     │     ├── ③ 生成执行计划（summary + detail + test strategy）     │     ├── ④ SandboxManager.CreateContainer()     │      └── 拉取 flowcode/sandbox:latest 镜像     │      └── 挂载 volume flowcode-sandbox-{issueId} → /workspace     │     ├── ⑤ git clone {project.gitRepoUrl} → /workspace/repo     │     ├── ⑥ 注入 Skills → /workspace/.flowcode_prompt.md     │      └── 内容: Issue 标题 + 描述 + 分析摘要 + 执行计划 + 绑定 Skill 指令     │     ├── ⑦ exec.CommandContext("opencode", "run",     │      "--prompt-file", "/workspace/.flowcode_prompt.md",     │      "--workspace", "/workspace/repo",     │      "--model", "claude-sonnet-4-20250514",     │      "--max-turns", "50")     │     ├── ⑥ stdout/stderr pipe → ctx.OnLog() → WebSocket → 前端实时日志     │     └── ⑦ cmd.Wait() → exitCode            ├── 0 → Transition("done")            │      → git add -A → git commit → git push            │      → create PR → Transition("pr_created")            └── ≠0 → Transition("fail") → retry 或通知管理员 ```  > **autoExecute 开关**：在项目设置中开启后，审批通过即自动入队，无需手动点击「执行」。适合个人项目或信任的自动化场景。
