RUN useradd --create-home --shell /bin/bash --uid 10001 sandbox \     && mkdir -p /workspace \     && chown -R sandbox:sandbox /workspace /home/sandbox  USER sandbox WORKDIR /workspace  ENTRYPOINT ["/usr/bin/tini", "--"] CMD ["sleep", "infinity"] ```  说明： - 当前仓库的 `Dockerfile.sandbox` 采用 `node:22-bookworm-slim`，优先保证 Node.js 版本稳定，同时控制镜像体积。 - Node.js 生态默认可用 `node`、`npm`、`npx`，并通过 `corepack` 支持 `yarn` / `pnpm`。 - 当前仓库默认在基础镜像中预装 `opencode`。 - `codex`、`claude-code` 暂未直接安装，先以 Dockerfile 注释占位，后续按官方安装方式补充。  ```bash # 构建 docker build -f Dockerfile.sandbox -t flowcode/sandbox:latest . ```  #### 语言专用镜像（通过 `config/sandbox.yaml` 项目覆盖）  不同语言项目通过 `overrides.image` 指定预装对应运行时的镜像：  ```yaml # config/sandbox.yaml overrides:   "java-project":     image: "flowcode/sandbox:java-21"     # 预装 JDK 21 + Maven + Gradle   "rust-project":     image: "flowcode/sandbox:rust-1.85"   # 预装 Rust + Cargo   "dotnet-project":     image: "flowcode/sandbox:dotnet-8"    # 预装 .NET SDK 8 ```  ```dockerfile # Dockerfile.sandbox.java — Java 专用镜像 FROM ubuntu:22.04 RUN apt-get update && apt-get install -y --no-install-recommends \     git curl wget ca-certificates openjdk-21-jdk maven \     nodejs npm python3 python3-pip golang-go \     && rm -rf /var/lib/apt/lists/* RUN curl -fsSL https://opencode.ai/install.sh | bash RUN useradd -m -s /bin/bash sandbox USER sandbox WORKDIR /workspace ```  > **原理**：TestCase.framework 决定执行命令，项目级 sandbox 镜像决定运行环境。两者解耦——同一框架（如 pytest）可运行在任何预装了 Python 的镜像中。
