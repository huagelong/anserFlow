[cosmtrek/air](https://github.com/cosmtrek/air) 实现代码变更自动重编译重启，避免每次修改后手动 `go build && ./flowcode server`：  ``` 开发工作流:   保存 .go 文件       │       ▼   air 检测文件变更       │       ▼   自动 go build → 杀死旧进程 → 启动新进程       │       ▼   浏览器刷新即可看到变更（< 3 秒） ```  `.air.toml` 配置：  ```toml # .air.toml root = "." tmp_dir = "tmp"  [build]   cmd = "go build -o ./tmp/flowcode ./cmd/server"   bin = "./tmp/flowcode server"   full_bin = ""   include_ext = ["go", "yaml", "yml", "toml", "json"]   exclude_dir = ["tmp", "docs", "web", "node_modules", ".git"]   include_dir = []   delay = 500          # ms，防抖延迟   stop_on_error = true # 编译失败时不重启  [log]   time = true  [misc]   clean_on_exit = true ```  启动方式：  ```bash # 安装 air go install github.com/cosmtrek/air@latest  # 开发模式启动（自动热加载） air  # 等价于： make dev ```  | 配置项 | 值 | 说明 | |--------|-----|------| | `delay` | 500ms | 防抖，避免连续保存触发多次重启 | | `stop_on_error` | true | 编译错误不杀旧进程，保持服务可用 | | `include_ext` | go/yaml/toml/json | 配置变更也触发重启 | | `exclude_dir` | tmp/docs/web | 排除无关目录，减少误触发 |  #### 2.3.7b 日志系统（slog）  Go 1.21+ 标准库 `log/slog` — 结构化日志，零第三方依赖：  **架构**：  ``` ┌─ 请求进入 ────────────────────────────────────────────┐ │                                                         │ │  Gin Middleware (Logger)                                │ │    ↓                                                    │ │  slog.Info("request",                                   │ │    "method", c.Request.Method,                         │ │    "path",   c.Request.URL.Path,                       │ │    "status", c.Writer.Status(),                        │ │    "latency", latency,                                  │ │    "ip",     c.ClientIP(),                             │ │  )                                                      │ │    ↓                                                    │ │  Handler / Service / FSM → 各自用 slog 记录关键事件      │ │    ↓                                                    │ │  slog.Error("issue execute failed",                     │ │    "issueId", issue.ID,                                │ │    "error",   err,                                     │ │  )                                                      │ │                                                         │ └─────────────────────────────────────────────────────────┘                            │               ┌────────────┼────────────┐               ▼            ▼            ▼          stdout       JSON 文件    日志采集系统        (开发 text)   (生产 json)  (可选: Loki/ELK) ```  **两种输出模式**：  | 模式 | 格式 | 配置 | 适用 | |------|------|------|------| | text | `time=... level=INFO msg="request" method=GET path=/api/...` | `log.format: text` | 开发，人眼友好 | | json | `{"time":"...","level":"INFO","msg":"request","method":"GET",...}` | `log.format: json` | 生产，日志采集 |  **日志级别**：`debug` → `info` → `warn` → `error`，环境变量 `LOG_LEVEL=debug` 可覆盖。  **关键日志点**：  | 位置 | 级别 | 内容 | |------|------|------| | Gin 中间件 | INFO | 每个 HTTP 请求: method / path / status / latency | | FSM 回调 | INFO | 状态转换: from / to / event | | FSM 回调 | WARN | 回调被取消 (CancelledError) | | Asynq Worker | INFO | 任务拾取 / 完成 / 失败 | | Asynq Worker | ERROR | 执行失败详情 + stacktrace | | HookRegistry | INFO | Webhook 发送成功 / 失败 | | GitService | INFO | clone / commit / push / create PR | | Docker Sandbox | DEBUG | 容器创建 / exec / 销毁 |  #### 2.3.7c 数据库迁移（Gorm AutoMigrate）  使用 Gorm 的 `AutoMigrate` 根据 Go struct 自动同步数据库 schema，无需手写 SQL migration：  **策略**：  ``` ┌─ 服务启动 ──────────────────────────────────────────┐ │                                                       │ │  1. 读取 FLOWCODE_DB_DRIVER (sqlite / mysql)          │ │  2. Gorm 连接对应数据库                                │ │     ├── sqlite: flowcode.db (当前目录, 自动创建)       │ │     └── mysql:  FLOWCODE_MYSQL_DSN 指定连接串          │ │  3. AutoMigrate(&User{}, &Org{}, &Project{}, ...)    │ │  4. 自动: CREATE TABLE / ADD COLUMN / ADD INDEX      │ │  5. 打印迁移日志: "migrated table: users"            │ │  6. 启动 Gin Server                                   │ │                                                       │ └───────────────────────────────────────────────────────┘ ```  **迁移模型注册**（启动时一次性执行所有 Model）：  ```go // internal/database/migrate.go func AutoMigrate(db *gorm.DB) error {     return db.AutoMigrate(         &model.User{},         &model.Organization{},         &model.OrgMember{},         &model.Project{},         &model.ApiKey{},         &model.GitCredential{},         &model.Requirement{},         &model.Issue{},         &model.Comment{},         &model.Label{},         &model.IssueLabel{},         &model.ExecutionLog{},         &model.TestCase{},         &model.TestResult{},         &model.HistoryRecord{},         &model.Skill{},         &model.AIToolConfig{},         &model.MonitorEvent{},         &model.Device{},         &model.AuditLog{},         &model.LoginLog{},     ) } ```  **AutoMigrate 行为**：  | 操作 | AutoMigrate 是否支持 | 说明 | |------|:---:|------| | 新建表 | ✅ | 根据 struct tag 创建 | | 新增字段 | ✅ | 自动 ADD COLUMN | | 新增索引 | ✅ | 自动 ADD INDEX | | 修改字段类型 | ❌ | 需手动 SQL 或 [Atlas](https://atlasgo.io/) 等工具 | | 删除字段 | ❌ | 不会 DROP COLUMN（安全保护） | | 重命名字段 | ❌ | 需手动迁移 |  > **生产环境建议**：当前默认部署仍可在启动时执行 AutoMigrate，以降低首次接入成本；但在需要严格变更审计的生产环境中，建议通过 `AUTO_MIGRATE=false` 关闭自动迁移，并搭配 [Atlas](https://atlasgo.io/) 或 [golang-migrate](https://github.com/golang-migrate/migrate) 维护版本化 DDL 脚本，实现可审查、可回滚的 migration。  **版本化迁移（后续升级）**：  ``` migrations/ ├── 001_create_users.up.sql ├── 001_create_users.down.sql ├── 002_create_organizations.up.sql ├── 002_create_organizations.down.sql ├── ... └── atlas.hcl                  # Atlas 配置 ```  开发期可直接使用 AutoMigrate；进入正式生产发布后，建议在发版前用 Atlas 导出 DDL → 生成版本化 migration 文件 → 提交 Git，并关闭自动迁移。  #### 2.3.7d SQLite 与 MySQL 差异  | 特性 | SQLite | MySQL | |------|--------|-------| | 安装 | 零配置，文件即数据库 | 需安装 MySQL Server | | 并发写入 | 单写者（WAL 模式缓解） | 多写者，高并发 | | JSON 查询 | GORM `Serializer` 模拟 | 原生 JSON 类型 + 函数 | | 全文搜索 | FTS5 扩展 | 原生 FULLTEXT 索引 | | 网络访问 | 不支持 | TCP 3306 | | 适用场景 | 单机 / 开发 / CLI 工具 | 多实例 / 生产 / 团队 |  **GORM 驱动切换**：  ```go // internal/database/database.go import (     "gorm.io/driver/sqlite"     "gorm.io/driver/mysql" )  func NewDB(cfg Config) (*gorm.DB, error) {     switch cfg.Driver {     case "sqlite":         return gorm.Open(sqlite.Open(cfg.SQLitePath), &gorm.Config{})     case "mysql":         return gorm.Open(mysql.Open(cfg.MySQLDSN), &gorm.Config{})     } } ```  > **JSON 兼容处理**：Issue 的 `tags`、`aiConfig`、`skillIds` 等 JSON 字段在 GORM 层通过 `gorm:"serializer:json"` tag 统一处理，SQLite 下自动转为 TEXT 存储，MySQL 下使用原生 JSON 类型。对上层业务代码透明。
