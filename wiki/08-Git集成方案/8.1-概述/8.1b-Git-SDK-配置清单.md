AI 编程工具集成](../../07-AI编程工具集成/README.md) | [10 - 部署与运维](../../10-部署与运维/README.md) ### 8.1b Git SDK 配置清单  #### 项目 Git 地址  | 字段 | 来源 | 说明 | |------|------|------| | `gitRepoUrl` | Project 表 | 仓库地址，支持 `https://` 和 `git@` 两种协议 | | `gitCredentialId` | Project 表 FK→GitCredential | **项目选用的凭证**，从已配置的凭证列表中选取 |  > 一个项目绑定一个仓库地址。先配置凭证（`POST git/connect`），再在项目设置中选择要使用的凭证。创建项目时可暂不选凭证，后续通过更新接口绑定。  #### Git 凭证（GitCredential 表）  一个项目可配置多条凭证（如对接多个仓库、多个平台），每条凭证通过 `name` 区分。支持 HTTP（PAT/OAuth）和 SSH 两种认证方式：  | 配置项 | HTTP 模式 | SSH 模式 | 说明 | |--------|:--------:|:--------:|------| | `name` | ✅ | ✅ | 凭证名称，如 "生产仓库"、"GitLab 备份" | | `provider` | ✅ | ✅ | `github` / `gitlab` / `gitea` | | `authMethod` | `http` | `ssh` | 认证方式 | | `accessToken` | ✅ | — | PAT 或 OAuth Token，AES-256-GCM 加密存储 | | `tokenType` | `pat` / `oauth` | — | Token 类型 | | `sshPrivateKey` | — | ✅ | SSH 私钥，AES-256-GCM 加密存储 | | `sshPassphrase` | — | 可选 | 私钥密码短语，加密存储 | | `expiresAt` | ✅ | ✅ | 过期时间 | | `lastUsedAt` | ✅ | ✅ | 最后使用时间（自动更新） |  #### GitProvider 工厂参数  ```go // internal/adapter/git/factory.go  const (     ProviderGitHub = "github"  // go-github SDK，无需 baseURL     ProviderGitLab = "gitlab"  // go-gitlab SDK，支持自建实例 baseURL     ProviderGitea  = "gitea"   // go-gitea SDK，支持自建实例 baseURL )  func NewGitProvider(pt ProviderType, accessToken, baseURL string) (GitProvider, error) ```  | 平台 | SDK | baseURL 必填 | 示例 | |------|-----|:---:|------| | GitHub | go-github | 否 | `NewGitHubProvider("ghp_xxx")` | | GitLab | go-gitlab | 自建时必填 | `NewGitLabProvider("glpat-xxx", "https://gitlab.example.com")` | | Gitea | go-gitea | 自建时必填 | `NewGiteaProvider("token", "https://gitea.example.com")` |  > GitHub 始终使用 `api.github.com`；GitLab/Gitea 的 SaaS 版（gitlab.com / gitea.com）baseURL 传空字符串即可，SDK 自动使用默认 API 端点。  #### 配置方式  | 方式 | 命令 / 端点 | |------|------------| | CLI | `flowcode project git-connect --name "生产仓库" --provider github --token ghp_xxx` | | CLI | `flowcode project set-credential --id cred_xxx`（为当前项目选择凭证） | | API | `POST /api/v1/projects/:pid/git/connect`（新增凭证） | | API | `GET /api/v1/projects/:pid/git/credentials`（凭证列表） | | API | `DELETE /api/v1/projects/:pid/git/credentials/:id`（删除凭证） | | API | `PUT /api/v1/projects/:id`（更新 `gitCredentialId` 选择凭证） | | Web | 项目设置 → Git 连接 → 选择平台 → 粘贴 PAT / SSH 密钥 |  **API 请求示例**：  ```bash # HTTP 方式（PAT）—— 新增凭证 curl -X POST /api/v1/projects/proj_1/git/connect \   -H "Authorization: Bearer $TOKEN" \   -d '{     "name": "GitHub 主仓库",     "authMethod": "http",     "accessToken": "ghp_xxxxxxxxxxxxxxxxxxxx",     "tokenType": "pat",     "provider": "github"   }'  # SSH 方式 curl -X POST /api/v1/projects/proj_1/git/connect \   -H "Authorization: Bearer $TOKEN" \   -d '{     "name": "公司 GitLab",     "authMethod": "ssh",     "sshPrivateKey": "-----BEGIN OPENSSH PRIVATE KEY-----\n...\n-----END OPENSSH PRIVATE KEY-----",     "provider": "gitlab"   }'  # 查看凭证列表 curl -X GET /api/v1/projects/proj_1/git/credentials \   -H "Authorization: Bearer $TOKEN"  # 删除凭证 curl -X DELETE /api/v1/projects/proj_1/git/credentials/cred_1 \   -H "Authorization: Bearer $TOKEN" ```
