TargetID   string    `gorm:"type:varchar(36);index" json:"targetId"`     Detail     string    `gorm:"type:varchar(500)" json:"detail"`       // 人类可读描述     Changes    JSONB     `gorm:"type:json" json:"changes,omitempty"`     // {field: {old, new}}     Source     string    `gorm:"type:varchar(20)" json:"source"`        // cli / web / api / webhook     DeviceID   string    `gorm:"type:varchar(64)" json:"deviceId,omitempty"`     IP         string    `gorm:"type:varchar(45)" json:"ip"`     CreatedAt  time.Time `gorm:"autoCreateTime" json:"createdAt"` } ```  ```sql CREATE TABLE audit_logs (     id VARCHAR(36) PRIMARY KEY,     user_id VARCHAR(36) NOT NULL,     user_name VARCHAR(100),     org_id VARCHAR(36) NOT NULL,     project_id VARCHAR(36),     action VARCHAR(50) NOT NULL,     target_type VARCHAR(30),     target_id VARCHAR(36),     detail VARCHAR(500),     changes JSON,     source VARCHAR(20) NOT NULL,          -- cli / web / api / webhook     device_id VARCHAR(64),     ip VARCHAR(45) NOT NULL,     created_at DATETIME NOT NULL,     INDEX idx_audit_user (user_id, created_at DESC),     INDEX idx_audit_org (org_id, created_at DESC),     INDEX idx_audit_project (project_id, created_at DESC),     INDEX idx_audit_action (action, created_at DESC),     INDEX idx_audit_target (target_type, target_id, created_at DESC) ); ```  **Action 枚举**：  | action | 说明 | targetType | |--------|------|------------| | `login.success` | 登录成功 | — | | `login.failed` | 登录失败 | — | | `issue.create` | 创建 Issue | issue | | `issue.update` | 更新 Issue | issue | | `issue.delete` | 删除 Issue | issue | | `issue.approve` | 审核通过 | issue | | `issue.reject` | 驳回 Issue | issue | | `issue.execute` | 触发 AI 执行 | issue | | `issue.cancel` | 取消执行 | issue | | `requirement.create` | 创建需求 | requirement | | `requirement.convert` | 需求拆解为 Issue | requirement | | `test_case.create` | 创建测试用例 | test_case | | `test_case.run` | 运行测试 | test_case | | `test_case.approve` | 审核通过测试用例 | test_case | | `skill.publish` | 发布 Skill | skill | | `skill.sync` | 同步 Skill 到 Git | skill | | `project.update` | 更新项目配置 | project | | `git.connect` | 配置 Git 凭证 | project | | `git.pr_create` | 创建 PR | issue | | `apikey.create` | 创建 API Key | api_key | | `apikey.revoke` | 吊销 API Key | api_key | | `member.invite` | 邀请成员 | org | | `member.remove` | 移除成员 | org |  > 登录日志独立存储于 `login_logs` 表（高频写入、低价值审计）；`audit_logs` 仅记录业务敏感操作，不重复存储登录事件。
