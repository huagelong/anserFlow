# AI 问询、执行日志与历史 Diff 设计

覆盖页面：

- `/projects/:pid/issues/:iid/questions`：IssueQuestions.vue
- `/projects/:pid/issues/:iid/history`：History.vue
- `/projects/:pid/issues/:iid/history/diff/:hid`：HistoryDiff.vue
- IssueDetail.vue 内的执行日志组件

## AI 主动问询页

### 页面目标

让用户及时回答 AI 在执行中提出的歧义问题，帮助任务继续运行。页面突出 pending 问题，不把已回答历史和待回答问题混在一起。

### 布局

标题栏：

- 标题：AI 问询。
- 副标题：显示 Issue 标题和当前状态。
- 右侧：返回 Issue 详情。

内容布局：

- 左侧问询列表，宽 360px。
- 右侧问询详情和回答区，填充剩余空间。
- 移动端变为上下结构。

### 问询列表

分组：

1. 待回答。
2. 已回答。
3. 已跳过 / 超时。

列表项：

- 问题摘要，两行。
- 状态徽章。
- askedAt。
- options 数量。

pending 项背景 `#FDF6EC`，左侧 3px 警告色。

### 问询详情

内容：

- AI 提问，Markdown 渲染。
- 上下文 context，代码或路径用等宽字体。
- 可选答案 options。
- 原始执行日志关联入口。

回答区：

- 如果有 options，使用单选卡片，卡片高度至少 44px。
- 下方提供“补充说明”文本域。
- 如果无 options，直接显示文本域。

动作：

- 主按钮：提交回答。
- 次按钮：跳过，让 AI 自行决策。

跳过需要确认，确认弹窗允许填写原因。

### 已回答状态

已回答项只读展示：

- selectedOption。
- answer。
- answeredBy。
- answeredAt。

不提供编辑回答，避免已恢复执行后的上下文不一致。

## 执行日志组件

### 目标

实时展示 AI 工具 stdout/stderr，帮助用户判断执行进度、失败原因和是否出现问询。

### 容器

最小高度 280px，可全屏。深色背景 `#111827`。

顶部工具栏高度 40px：

- 连接状态：连接中 / 实时连接 / 已断开 / 重连中。
- 当前工具：opencode / codex / claude-code。
- attempt。
- 日志级别筛选：all / stdout / stderr / question。
- 自动滚动 Switch。
- 复制当前日志。
- 查看完整 stdout。

### 日志行

行高 20px，字号 13px。

结构：

- 时间戳，弱色。
- stream 标识。
- 内容。

颜色：

- stdout：`#DCDFE6`。
- stderr：`#F56C6C`。
- question marker：`#E6A23C` 背景弱高亮。
- success：`#67C23A`。

### WebSocket 状态

- 首次连接：显示 skeleton 终端行。
- 断开：顶部显示黄色连接条，不清空内容。
- 重连成功：toast 提示，继续追加日志。
- 执行结束：显示结束摘要，包含 exitCode、filesChanged、耗时。

### 完整 stdout 视图

点击“查看完整 stdout”打开抽屉，宽 70vw，最大 1100px。

功能：

- 搜索。
- 复制。
- 显示日志大小，超长时虚拟滚动。

## 历史记录列表

### 页面目标

查看 Issue 或需求的字段级变更历史，定位谁在什么时候做了什么。

### 布局

左侧时间线，右侧详情摘要。

列表项：

- action 类型。
- 操作人。
- 来源：web / cli / api / webhook。
- targetType。
- createdAt。
- 变更摘要。
- 操作：查看 diff。

筛选：

- 操作类型。
- 来源。
- 操作人。
- 时间范围。

### 时间线视觉

- 时间线轴颜色 `#DCDFE6`。
- 当前选中项背景 `#ECF5FF`。
- 危险操作使用危险色图标。

## Diff 页面

### 页面目标

展示字段级双栏 diff，支持回滚确认。

### 布局

顶部：

- 面包屑。
- 变更标题。
- 操作人、时间、来源。
- 返回历史。

主体：

- 左栏：变更前。
- 右栏：变更后。
- 使用 Monaco 或 diff2html 渲染。
- 每个字段作为独立区块。

尺寸：

- 左右栏各 50%。
- 栏间距 16px。
- 代码字体 13px。

### 回滚

如果该历史记录支持回滚：

- 右上角显示“回滚到此版本”。
- 点击后弹窗显示将回滚的字段列表。
- 必须确认后执行。

不支持回滚时：

- 显示禁用按钮并提示原因，如“执行日志、审计记录不可回滚”。

### 空态与错误态

- 无历史：显示空态，引导回到详情。
- diff 数据缺失：显示“无法加载该版本内容”，提供重试。

## 交互补充

- AI 问询页打开时默认选中第一条 pending 问题；没有 pending 时选中最近一条问题。
- 提交回答成功后，当前问题从待回答分组移动到已回答分组，列表不刷新到顶部。
- 跳过问询前显示确认弹窗，说明 AI 将自行决策并继续执行。
- 日志自动滚动开启时，新日志到达自动贴底；用户手动向上滚动后自动关闭贴底并显示“回到底部”按钮。
- 历史 diff 页面从列表进入后，返回时保留历史列表的筛选条件和滚动位置。
