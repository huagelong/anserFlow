# 需求管理页面设计

覆盖页面：

- `/projects/:pid/requirements`：Requirements.vue
- `/projects/:pid/requirements/:rid`：RequirementDetail.vue

## 需求列表

### 页面目标

管理自然语言需求，支持从需求进入 AI 拆解，并将确认后的拆解结果批量转为 Issue。

### 页面结构

标题栏：

- 标题：需求。
- 副标题：展示当前项目名。
- 主按钮：新建需求。

工具栏：

- 搜索：标题、描述。
- 状态筛选：draft / reviewing / approved / in_progress / done / archived。
- 优先级筛选：p0 / p1 / p2 / p3。
- 来源筛选：manual / monitor / api / cli。

列表使用表格。

### 表格列

| 列 | 宽度 | 内容 |
|----|------|------|
| 需求 | 360px | 标题、描述摘要、来源。 |
| 状态 | 120px | 状态徽章。 |
| 优先级 | 96px | p0-p3 徽章。 |
| 拆解 | 140px | 已拆解 / 未拆解 / 拆解中。 |
| 创建者 | 140px | 头像 + 名称。 |
| 更新时间 | 160px | 相对时间。 |
| 操作 | 160px | 查看、编辑、删除。 |

### 新建需求

使用右侧抽屉，宽 640px。

字段：

- 标题，必填，最多 500 字。
- 描述，Markdown，最小高度 240px。
- 优先级，单选。
- 来源由系统写入，不允许手动选择。

保存后进入详情页。

### 状态表现

| 状态 | 颜色 | 下一步 |
|------|------|--------|
| draft | 灰色 | 编辑、提交审核。 |
| reviewing | 黄色 | 等待审核。 |
| approved | 蓝色 | AI 拆解。 |
| in_progress | 青色 | 查看关联 Issue。 |
| done | 绿色 | 归档。 |
| archived | 灰色 | 只读查看。 |

## 需求详情

### 页面目标

让用户阅读需求、查看拆解结果、调整并确认生成 Issue。

### 布局

桌面端两栏：

- 左侧主内容：需求正文、拆解结果、关联 Issue。
- 右侧状态栏：需求状态、优先级、创建者、操作历史摘要。

左侧宽度 `calc(100% - 384px)`，右侧 360px，间距 24px。

### 详情头部

内容：

- 标题。
- 状态、优先级、来源。
- 创建者、创建时间、更新时间。
- 操作：编辑、提交审核、AI 拆解、删除。

动作展示必须遵循状态：

- 未 approved 不显示 AI 拆解主按钮。
- archived 只读。

### 需求正文

Markdown 渲染：

- 最大宽度 820px。
- 标题、列表、代码块样式与 Issue 描述一致。
- 右上角“编辑”按钮进入抽屉。

### AI 拆解面板

触发条件：

- Requirement.status 为 `approved`。
- 用户点击“AI 拆解”。

加载态：

- 面板显示步骤：准备上下文 → 调用 opencode → 解析 JSON → 返回建议。
- 不展示虚假的进度百分比。

结果结构：

- 摘要：AI 分析需要几个 Issue。
- Issue 建议列表。

每条建议卡片：

| 区域 | 内容 |
|------|------|
| 左侧 | Checkbox，默认选中。 |
| 标题 | 可编辑输入框。 |
| 描述 | 可展开 Markdown 文本域。 |
| 元信息 | priority、category、aiToolHint。 |
| 理由 | AI 拆解 reason，弱文字显示。 |

可调整项：

- 勾选 / 取消勾选。
- 修改标题。
- 修改优先级。
- 修改分类。
- 修改 AI 工具。

不提供未定义的依赖编排编辑器，只显示 AI reason 中的顺序建议。

### 确认拆解

底部固定操作栏：

- 左侧：已选择 N 个 Issue。
- 次按钮：重新拆解。
- 主按钮：确认创建 Issue。

确认时调用 `POST /api/v1/requirements/:id/convert-to-issues`。

成功后：

- 状态转为 in_progress。
- 显示关联 Issue 列表。
- 提供跳转 Issue 看板。

### 关联 Issue 区

列表展示由需求拆解生成或手动关联的 Issue：

- 标题。
- 状态。
- 优先级。
- 分类。
- AI 工具。
- PR 状态。

点击进入 Issue 详情。

### 空态与错误态

- 未拆解：展示“需求审核通过后可拆解为 Issue”。
- 拆解失败：展示错误信息、重试按钮，不清空原需求。
- 解析结果为空：提示“AI 未返回可用 Issue”，允许重新拆解。

## 交互补充

- 需求列表的状态筛选与搜索条件在进入详情后保留；返回列表时恢复原状态。
- 创建需求成功后进入详情页，并在标题区显示“已创建，等待审核”的轻提示。
- AI 拆解运行中时，用户可收起拆解面板但不能再次触发拆解。
- 用户调整拆解结果后，离开详情或重新拆解前必须确认是否放弃当前调整。
- 确认创建 Issue 成功后，拆解结果卡片变为只读，并显示每个新 Issue 的跳转入口。
