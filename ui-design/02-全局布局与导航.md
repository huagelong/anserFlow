# 全局布局与导航

## 页面框架

Web 控制台采用固定顶栏 + 项目侧边栏 + 内容区的应用框架。

```
┌──────────────────────────────────────────────────────────────┐
│ Topbar 56px                                                   │
├──────────────┬───────────────────────────────────────────────┤
│ Sidebar 248  │ Content                                       │
│              │                                               │
│              │                                               │
└──────────────┴───────────────────────────────────────────────┘
```

## 顶栏

高度 56px，背景 `#FFFFFF`，底部边框 `#E5E7EB`。

左侧：

- flowcode 标识，宽 32px，高 32px。
- 当前组织切换器，宽 180-240px。
- 当前项目切换器，宽 220-280px，仅进入项目后显示。

中间：

- 预留项目状态摘要区域，显示当前项目的待审核 Issue、待回答 AI 问询、运行中任务数量。

右侧：

- AI 待回答问询入口，显示 pending 数。
- 通知入口，用于执行完成、失败、登录异常等。
- 语言切换：中文 / English。
- 用户菜单：个人资料、登录日志、设备管理、退出。

## 侧边栏

展开宽度 248px，收起宽度 72px。背景 `#FFFFFF`，右侧边框 `#E5E7EB`。

导航分组：

| 分组 | 项目 | 路由 |
|------|------|------|
| 工作区 | 仪表盘 | `/` |
| 工作区 | Issue 看板 | `/projects/:pid/issues` |
| 工作区 | 需求管理 | `/projects/:pid/requirements` |
| 工作区 | PR 管理 | `/projects/:pid/prs` |
| 工作区 | 测试用例 | `/projects/:pid/test-cases` |
| 工作区 | 监控面板 | `/projects/:pid/monitor` |
| 管理 | 项目设置 | `/projects/:pid/settings` |
| 管理 | Skills 管理 | `/projects/:pid/skills` |
| 管理 | 审计日志 | `/projects/:pid/audit-log` |
| 组织 | 组织管理 | `/org/:slug` |
| 组织 | 设备管理 | `/org/:slug/devices` |

交互：

- 当前路由背景 `#EFF6FF`，文字 `#2563EB`，左侧 3px 主色条。
- 有待处理的 AI 问询时，Issue 看板与 AI 问询入口显示黄色计数徽章。
- 侧边栏收起后仅显示图标，hover 显示 tooltip。
- 移动端侧边栏变为抽屉，从左侧滑入，宽度 280px。

## 内容区

内容区背景 `#F7F8FA`。默认 padding：桌面 24px，移动端 16px。

页面结构：

1. 面包屑：高度 24px，字号 13px。
2. 页面标题栏：高度 48-64px，包含标题、副标题、主操作。
3. 工具栏：筛选、搜索、视图切换。
4. 主内容：表格、看板、详情布局。

标题栏规范：

- 页面标题 20px/30px，字重 600。
- 副标题 13px/20px，颜色 `#6B7280`。
- 主操作按钮放右侧，移动端固定在标题下方。

## 项目上下文

进入项目后所有页面都必须明确显示当前项目：

- 顶栏项目切换器显示项目名、Git provider、小状态点。
- 内容页标题下可显示项目 Git 仓库、默认分支、最近验证时间。
- Git 未配置或验证失败时，在项目页顶部显示警告条，引导到项目设置，不阻断查看已有数据。

## 通知中心

通知类型只覆盖文档定义事件：

- AI 执行完成/失败。
- AI 主动询问待回答。
- Git 导入完成。
- PR 状态变化。
- 测试运行完成。
- 登录异常、新设备登录。

每条通知包含类型、标题、关联对象、时间和一个主操作。

## 权限表现

文档中明确“管理员”的动作，在 UI 上遵循：

- 无权限时隐藏危险或管理按钮。
- 如果动作对理解状态重要，显示禁用按钮并用 tooltip 说明“需要管理员权限”。
- 审计日志中仍显示用户自己的相关记录。

## 国际化

界面文案使用 vue-i18n 管理。设计稿以中文为主，但组件宽度需兼容英文：

- 按钮最小宽度 72px，长英文可换为图标 + tooltip。
- 表格列标题不写死宽度，使用 `min-width`。
- 状态徽章保留固定色，不依赖语言。

## 快捷操作入口

高频创建动作放在页面内，不做全局悬浮按钮：

- 项目列表：新建项目。
- 需求列表：新建需求。
- Issue 看板：新建 Issue、导入 Git Issues。
- 测试用例列表：从 Issue 详情创建测试用例。
- 监控事件列表：转 Issue。

## 加载与刷新

- 页面首次进入显示骨架屏。
- 顶栏项目切换后，内容区整体刷新，侧边栏保留。
- WebSocket 断开时，在相关日志面板显示连接条：`实时连接已断开，正在重连...`，并保留历史日志。

## 全局交互补充

- 顶栏组织或项目切换后，先关闭未保存的抽屉和弹窗；若存在未保存编辑内容，先触发离开确认。
- 通知入口只显示需要用户回流处理的事件，点击后进入对应详情页并定位到相关 Tab。
- 侧边栏收起状态下，当前页仍保留选中态；hover tooltip 展示完整名称。
- 权限不足的管理动作不进入更多菜单，避免用户误以为可通过隐藏入口完成。
