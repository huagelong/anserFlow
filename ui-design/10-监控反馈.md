# 监控反馈页面设计

覆盖页面：

- `/projects/:pid/monitor`：Monitor.vue

## 页面目标

展示轻量级问题回流管道：线上事件如何聚合、去重、分级，并转化为 Issue。页面不做完整 APM，只覆盖事件趋势、列表、详情和转 Issue。

## 顶部区域

标题栏：

- 标题：监控面板。
- 副标题：当前项目和时间范围。
- 右侧：时间范围选择，默认最近 7 天。

时间范围：

- 最近 24 小时。
- 最近 7 天。
- 最近 30 天。
- 自定义。

## 统计卡片

四张卡片，高 112px：

| 卡片 | 指标 | 说明 |
|------|------|------|
| 总事件 | total | 时间范围内事件总数。 |
| 已处理 | processed | 已关联或已处理事件。 |
| 自动创建 | issueCount | 自动创建 Issue 数。 |
| 平均响应 | avgResponseTime | 从事件到处理的平均时间。 |

卡片下方显示与上一周期相比的变化，只显示数值，不做预测。

## 趋势图

图表高度 280px。

内容：

- X 轴：时间。
- Y 轴：事件数量。
- 线条：error、warning、feedback 可用多色区分。
- Tooltip：时间、类型、数量。

空态：

- 时间范围内无事件时显示“暂无监控事件”。

## 事件列表

### 工具栏

- eventType 筛选：error / unhandledrejection / console.error / fetch-error / warning / perf / feedback。
- 状态筛选：未处理 / 已转 Issue / 已修复。
- 搜索：message、url、dedupHash。
- 环境筛选：production / staging / development，来自 metadata。

### 表格列

| 列 | 宽度 | 内容 |
|----|------|------|
| 时间 | 160px | 最近出现时间。 |
| 类型 | 140px | eventType 徽章。 |
| 消息 | 360px | message 摘要、url。 |
| 次数 | 96px | 聚合次数。 |
| 环境 | 120px | metadata.environment。 |
| 关联 Issue | 180px | Issue 链接或空。 |
| 操作 | 140px | 查看、转 Issue。 |

行样式：

- error 使用危险色徽章。
- warning 使用警告色徽章。
- feedback 使用主色徽章。

## 事件详情抽屉

宽 720px。

内容区：

1. 概览：eventType、message、source、次数、dedupHash。
2. metadata：url、method、statusCode、userAgent、userId、duration、environment、releaseVersion。
3. stackTrace：代码块展示。
4. 关联 Issue：已关联则显示链接和状态。
5. 操作记录。

动作：

- 转 Issue。
- 查看关联 Issue。

## 转 Issue

### 触发

事件未关联 Issue 时展示“转 Issue”。自动创建的事件显示已关联。

### 弹窗

宽 640px。

预填字段：

- 标题：来自事件消息。
- 描述：包含错误信息、堆栈、上下文、统计。
- 优先级：按文档优先级推断规则。
- 分类：bug。
- 标签：monitor、eventType。

用户可编辑：

- 标题。
- 描述。
- 优先级。
- AI 工具。

确认后调用 `POST /api/v1/monitor/events/:id/convert`。

成功后：

- 事件行显示关联 Issue。
- 打开 Issue 详情或留在列表。

## SDK 接入信息

页面底部提供接入说明区，作为项目开发者查看使用：

- Frontend SDK snippet。
- Server SDK 简要入口。
- HTTP API 地址。
- projectKey 只展示当前项目 key，不展示无关配置。

代码块使用等宽字体，提供复制按钮。

## 已修复追溯

如果事件关联的 Issue 已部署，且同 dedupHash 不再出现：

- 事件状态显示已修复。
- 详情中显示修复 Issue、PR、部署时间。

该状态由后端检测，UI 只展示，不新增手动“标记修复”功能。

## 空态与错误态

- 无 SDK 事件：显示接入 snippet 和“复制代码”。
- 查询失败：显示重试。
- 转 Issue 失败：保留弹窗内容，展示错误。
- 权限不足：转 Issue 按钮禁用。

## 交互补充

- 统计卡片点击后过滤事件列表，例如点击“自动创建”只展示已关联 Issue 的事件。
- 事件详情抽屉打开后，列表当前行保持选中态，关闭抽屉后回到原滚动位置。
- 转 Issue 弹窗预填内容后，用户编辑过的字段不再被事件刷新覆盖。
- 转 Issue 成功后，事件详情抽屉内立即显示关联 Issue，并提供打开 Issue 的主按钮。
- 复制 SDK 代码成功后显示短 toast；复制失败时保留代码块并提示手动复制。
